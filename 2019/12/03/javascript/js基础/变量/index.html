变量

阅读量: 静思 2019-12-03 10:18:12
Categories： javascript Tags： js基础

ECMAScript 变量是松散类型：变量可以用于保存任何类型数据。每个变量只不过是用于保存任意值的命名占位符。通过三个关键字声明变量：var、let、const。区别是 var 所有版本都可以使用；let、const 只能在 es6+版本使用

1. var 关键字

var message;
// 定义变量名为 message 的变量用于保存任意类型的值
// 未初始化默认保存一个特殊值 undefined
var message1 = "hi";
// 定义一个名为 message1 的变量并设置 String 类型的值为 "hi"
message1 = 100;
// 重写保存为 Number 类型的值为 100；虽然不推荐但是有效

1.1 var 声明作用域：

  1. var 定义的变量会成为包含它的函数的局部变量（函数作用域）；函数内部使用 var 定义的变量将在函数退出时销毁
    function test () {
    var message = "hi"; // 局部变量
}
test();
console.log(message); // 出错
  2. 函数内不使用 var 定义的变量称为隐式全局变量（可以在函数外部访问到）
    function test () {
    message = "hi"; // 局部变量
}
test();
console.log(message); // "hi"
    • 不推荐使用隐式全局变量
      • 增加维护难度，不能直接判断是否为全局变量
      • 严格模式下，给未声明的变量赋值会抛出 ReferenceError 错误
  3. 多个变量定义通过在单个语句中使用逗号分隔
    var message ="hi",
    found = false,
    age = 23;
  4. 严格模式下不能定义名为 eval 和 arguments 的变量，否则会导致语法错误

1.2 var 声明提升：

  1. var 声明的变量会自动提升到函数作用域顶部；这就是所谓的“提升”（hoist）
    function foo () {
    console.log(age);
    var age = 26;
}
foo(); // undefined
// 输出 undefined 原因可理解为下列代码
function foo () {
    var age;
    console.log(age);
    age = 26;
}
foo(); // undefined
  2. var 可以重复声明同一变量
    function foo () {
    var age = 16;
    var age = 26;
    var age = 36;
    console.log(age);
}
foo() // 36

2. let 声明：

  • 作用：let 和 var 差不多

  • 区别：

    • let 是块作用域；var 是函数作用域
    • let 禁止重复声明变量
    • let 不会变量声明提升
    • let 在全局作用域中声明的变量不会成为 window 对象的属性
      ```
      if (true) {
      var name = ‘Matt’;
      console.log(name); // Matt
      }
      console.log(name); // Matt

    if (true) {

      let age = 26;
  console.log(age); // 26

    }
    console.log(age); // ReferenceError: age is not defined
    ```

  • 适用于 var 的作用域也适用于 let

    • let 作用域仅限于当前块的内部
    • 块作用域是函数作用域的子集

  • let 禁止相同块作用域中出现冗余声明，导致报错

    let age;
let age;
// Uncaught SyntaxError: Identifier 'age' has already been declared
// 未捕获 的 语法错误：标识符 “变量名” 已经被声明

  • 嵌套使用相同标识符不会报错，因为相同块中没有重复声明

    • js 引擎会记录变量及其所在的块作用域
      var name = 'Nicholas';
console.log(name); // 'Nicholas'
if (true) {
  var name = 'Matt';
  console.log(name); // 'Matt'
}
let age = 30;
console.log(age); // 30
if (true) {
  let age = 26;
  console.log(age); // 26
}

  • 声明冗余报错不会因混用 var 和 let 受影响

    • 这两个关键字声明的不是不同类型的变量
    • 只是指出变量在相关作用域的存在方式
      let name;
var name; // SyntaxError

      2.1 暂时性死区

  • let 声明之前的执行瞬间被称为“暂时性死区”（temporal dead zone）,此阶段引用任何之后声明的变量都会抛出 ReferenceError

    // var 变量声明提升
console.log(name); // undefined
var name = 'Matt';
// let 不会变量声明提升
console.log(age); // ReferenceError age is not defined
let age = 18;

    2.2 全局声明

  • 与 var 关键字不同；let 在全局作用域中声明的变量不会成为 window 对象的属性

    var name = 'Matt';
console.log(wiindow.name); // 'Matt'

let age = 18;
console.log(window.age); // undefined

  • let 声明仍是在全局作用域中发生，相应变量会在页面生命周期内存续。因此，为避免 SyntaxError，必须确保页面不会重复声明同一变量

2.3 条件声明

  • var 声明变量时，因声明被提升，js 引擎会自动将多余声明在作用域顶部合并为一个声明

  • 因 let 是块作用域，所以不可能检查前面是否已经使用 let 声明过相同变量，就不可能在未声明的情况下声明它

    <script>
var name = 'Nicholas';
let age = 26;
</script>
<script>
// 假设脚本不确定页面是否已经声明相同变量；则可以假设未被声明
var name = 'Matt';
/*
 * 使用 var 声明的变量在变量声明提升时会将多余相同变量合并，
 * 因此不需要检查之前是否声明过同名变量
 */
let age = 36;
// 若 age 之前声明过，此处报错
</script>

  • 无法通过 try/catch 语句或 typeof 操作符解决；因 条件块中 let 声明的作用域仅限于该块

    <script>
var name = 'Nicholas';
let age = 26;
</script>
<script>
// 假设脚本不确定页面是否已经声明相同变量；则可以假设未被声明
if (typeof name === 'undefined') {
    let name;
}
// name 被限制在 if 块作用域内，因此下列赋值形同全局赋值
name = 'Matt';
try (age) {
 // 若 age 未声明则报错
}
catch (error) {
    let age;
}
// age 被限制在 catch 块作用域内，因此下列赋值形同全局赋值
age = 18;
</script>

  • let 这个 es6 新增声明关键字，不能依赖条件声明模式

    • 禁止通过条件声明使用 let 是好事，因条件声明式一种反模式，让程序更复杂

2.4 for 循环中的 let 声明

  • let 出现前，for 循环定义的迭代遍历会渗透到循环体外部
    for (var i = 0; i < 5; ++i) { /* 循环逻辑 */ }
console.log(i); // 5
  • 使用 let 解决此问题；因迭代遍历的作用域仅限于 for 循环内部
    for (let i = 0; i < 5; ++i) { /* 循环逻辑 */ }
console.log(i); // ReferenceError: i is not defined
  • 使用 var 时，最常见的问题是对迭代变量的奇特声明和修改
    for (var i = 0; i < 5; ++i) {
    setTimeout(() => console.log(i), 0) // 5、5、5、5、5
}
    • 此问题原因是：
      • 退出循环时，迭代变量保存的时导致循环退出的值
      • 之后执行超时逻辑时，所有 i 都是相同变量，因此输出都是同一个最终值
  • 使用 let 声明迭代变量时， js 引擎在后台会为每个迭代循环声明一个新的迭代变量，每个 setTimeout 引用的都是不同的变量实例，所以输出的是我们的预期，即：循环执行过程中每个迭代变量的值；此迭代声明一个独立变量实例的行为适用于所有风格的 for 循环，包括 for-in 和 for-of
    for (let i = 0; i< 5; ++i) {
    setTimeout(() => console.log(i), 0) // 0、1、2、3、4
}

3. const 声明

  • 区别：

    • const 声明变量时必须同时初始化变量，且修改 const 声明的变量会导致运行报错
    • const 禁止重复声明
    • const 是块作用域
      ```
      const age = 18l
      age = 20; // TypeError: 给常量赋值

    const name = ‘Matt’;
    const name = ‘Nicholas’; // SyntaxError

    // const 是块作用域
    const name = ‘Matt’;
    if (true) {

      const name = 'Nicholas';

    }
    console.log(name); // Matt
    ```

  • const 变量引用的是一个对象，则修改这个对象内部的属性并不违反 const 的限制

    const person = {};
person.name = 'Matt'; // ok

  • js 引擎为 for 循环中 let 声明分别创建独立的变量实例，虽然 const 和 let 相似，但不能用 const 声明迭代变量（迭代变量会自增）

    for (const i = 0; i < 5; ++i) {} // TypeofError：给常量赋值

  • 可以使用 const 声明一个不被修改的 for 循环变量。每次迭代只创建新变量。这对 for-of 和 for-in 特别有意义

    let i = 0;
for (const j = 7; i < 5; ++i) {
    console.log(j); // 7, 7, 7, 7, 7
}

for (const key in {a: 1, b: 2}) {
    console.log(key); // a, b
}

for (const value of [1, 2, 3, 4, 5]) {
    console.log(value); // 1, 2, 3, 4, 5
}

4. 声明风格及最佳实践

  1. 不使用 var 定义变量
    • 使用新关键字定义变量明确作用域、声明位置、不变的值（常量）
  2. const 优先、let 次之
    • 优先使用 const 可以使浏览器运行时强制保持变量不变，也可让静态代码分析工具提前发现不合法的赋值操作
    • 开发者更有信心推断某些变量的值永远不变，同时也能迅速发现因意外赋值导致的非预期行为

小结：

  • 变量定义方式：

    • var
    • let
    • const

  • 区别：

    • var 是函数作用域；let、const 是块作用域（函数作用域的子集）
    • var 允许重复声明；let、const 禁止
    • var 变量声明提升；let、const 没有
    • var 全局作用域中声明的变量可以通过全局对象 window 找到；let、const 全局作用域中声明的全局对象不可以通过 window 找到
    • var、let 声明的是变量其值可以修改；const 声明的是常量，声明时需同时初始化值，且一经定义不可修改；但是 const 声明的是一个对象时，该对象内的属性可以修改
