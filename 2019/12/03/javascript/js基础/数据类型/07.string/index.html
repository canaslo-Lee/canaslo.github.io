07.string

阅读量: 静思 2019-12-03 10:18:12
Categories： javascript Tags： js基础

描述

  • 表示零或多个16位unicode字符序列化
  • 字符串可使用双引号、单引号、反引号标示
  • 字符串引号开头和结尾必须一致

1. 字符字面量

  • string类型包含字符字面量，用于表示非打印字符或其他用途字符
  • 这些字符字面量可出现再字符串中的任意位置且可作为单个字符被解释
  • 如下：
    字面量 含义
    \n 换行
    \t 制表
    \b 退格
    \r 回车
    \f 换页
    \ 反斜杠
    ' 单引号
    " 双引号
    ` 反引号
    \xnn 十六进制编码nn表示的字符（n是十六进制数字0~F）
    \unnnn 十六进制编码nnnn表示的unicode字符（n是十六进制数字0~F）
  • 字符串长度可通过length属性获取；若字符串中包含双字节符号，返回的值可能不是准确字符数
    let text = "this is the letter sigma"
console.log(text.length) // 24

2. 特点

  • es中字符串不可变（immutable），一旦创建值就不能改变了
  • 若要修改变量中的字符串值，必须先销毁原始的字符串，然后将包含新值的另一个字符串保存到该变量

3. 转为字符串

  • toString()
    • 作用：
      • 返回当前值的字符串等价物
    • 拥有该方法的数据类型：
      • 数值、布尔值、对象、字符串值
    • 只有在数值调用该方法时接收一个底数参数（以什么底数输出数值的字符串表示）
    • 默认返回数值的十进制字符串表示；
      • 通过传入参数可获取二进制、八进制、十六进制、或其他任何有效基数的字符串表示
    • 默认（不传参数）的输出与传入参数10得到的结果一致
  • String()
    • 不确定值是否为null或undefined时使用
    • 始终返回表示相应类型值的字符串
    • 规则：
      • 值有toString()方法，则调用该方法（不传参数）并返回结果
      • 值为null 返回 “null”
      • 值为undefined 返回 “undefined”
  • 加号操作符给值加空字符串
    let num = 123
let str = num + ""
console.log(str, typeof str) // 123 string

4. 模板字面量

  • es6新增，与使用单双引号不同，模板字面量保留换行字符，可跨行定义字符串
  • 由于模板字面量会保持反引号内部空格，因此使用时需注意
  • 格式正确的模板字符穿可能看起来缩进不当

5. 字符串插值

  • 模板字面量最常用的一个特性时支持字符串插值；即可在一个连续定义插入一个或多个值
  • 技术角度模板字面量不是字符串，而是特殊的js句法表达式，求值之后得到的时字符串
  • 模板字面量在定义时立即求值并转换为字符串实例，任何插入的变量也会从其最接近的作用域中取值
  • 通过 ${}中js表达式实现
    let val = 5, exponent = 'second';
let interpolatedTemplateLiteral = `${ val } to the ${ exponent } power is ${ val * val}`
console.log(interpolatedTemplateLiteral) // 5 to the second power is 25
  • 所有插入值都使用toString()强制转为字符串且任何js表达式都可用于插值；嵌套的模板字符串无需转义
  • 插值表达式中可调用函数和方法
    function capitalize(word) {
  return `${word[0].toUpperCase()}${word.slice(1)}`
}
console.log(`${capitalize('hello')}, ${capitalize('world')}!`) // Hello, World!
  • 可插入自己之前的值
    let val = '';
function append() {
  val = `${val}abc`
  console.log(val)
}
append() //abc
append() // abcabc

6. 模板字面量标签函数

  • 模板字面量也支持定义标签函数（tag function），通过标签函数可自定义插值行为

  • 标签函数会接收被插值记号分隔后的模板和对每个表达式求值的结果

  • 标签函数本身时常规函数，通过前缀到模板字面量来应用自定义行为

    /**
* 标签函数接收到的参数一次是原始字符串数组和对每个表达式求值的结果
* 该函数的返回值是对模板字面量求值得到的字符串
*/
let a = 6, b = 9;
function simpleTag(strings, aValExp, bValExp, sumExp) {
  console.log(strings) // ["", " + ", " = ", ""]
  console.log(aValExp) // 6
  console.log(bValExp) // 9
  console.log(sumExp)  // 15
  return 'foobar'
}
let untaggedResult = `${ a } + ${ b } = ${ a + b }`,
    taggedResult = simpleTag`${ a } + ${ b } = ${ a + b }`;
console.log(untaggedResult) // "6 + 9 = 15"
console.log(taggedResult)   // "foobar"

// 表达式参数的数量可变，通常应使用剩余操作符（rest operator）将其收集到一个数组中
let a = 6, b = 9;
function simpleTag(strings, ...exps) {
  console.log(strings) // ["", " + ", " = ", ""]
  for (const exp of exps) console.log(exp);
  // 6
  // 9
  // 15
  return 'foobar'
}
let taggedResult = simpleTag`${ a } + ${ b } = ${ a + b }`
console.log(taggedResult) // "foobar"

  • 对有n个插值的模板字面量，传给标签函数的表达式参数个数始终是n

  • 传给标签函数第一个参数所包含的字符串个数始终是n+1

  • 如果想把该字符串和对表达式求值的结果拼接作为默认返回的字符串可参考下例

    let a = 6, b = 9;
function zipTag (strings, ...exps) {
  return strings[0] + exps.map((e, i) => `${ e }${ strings[i + 1] }`).join('')
}
let untaggedResult = `${ a } + ${ b } = ${ a + b }`,
    taggedResult = zipTag`${ a } + ${ b } = ${ a + b }`;
console.log(untaggedResult) // "6 + 9 = 15"
console.log(taggedResult)   // "6 + 9 = 15"

7. 原始字符串

  • 使用模板字面量也可直接获取原始模板字面量内容；如：换行符、unicode字符；

  • 而不是被转换后的字符表示，可使用默认的 String.raw 标签函数

    // unicode demo
// \u00A9 是版权符号
console.log(`\u00A9`) // ©
console.log(String.raw`\u00A9`) // \u00A9

// 换行符 示例
console.log(`first line\nesecond line`)
// first line
// nesecond line
console.log(String.raw`first line\nesecond line`) // "first line\nesecond line"

/**
 * 对实际的换行符来说是不行的
 * 它们不会被转换为转义序列的形式
*/
console.log(`first line
second line`)
// first line
// second line
console.log(String.raw`first line
second line`)
// first line
// second line
/**
 * 可通过标签函数的第一个参数
 * 字符串数组的.raw属性取得每个字符串的原始内容
*/
function printRaw (strs) {
  console.log('Actual characters')
  for (const str of strs) console.log(str)
  console.log('Escaped characters')
  for (const rawStr of strs.raw) console.log(rawStr)
}
printRaw`\u00A9${'and'}\n`
// Actual characters
// ©
// （换行符）
// Escaped characters
// \u00A9
// \n
