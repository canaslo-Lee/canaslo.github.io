数据类型

阅读量: 静思 2019-12-03 10:18:12
Categories： javascript Tags： 基础

ECMAScript有==6种简单数据类型（原始类型）==

  • Undefined
  • Null
  • Boolean
  • Number
  • String
  • Symbol：Symbol（符号）es6新增

==一种复杂数据类型叫Object（对象）==

  • Object是一种无序名值对的集合。

注解：

  • 在ECMAScript中不能定义自己的数据类型，所有值都可以用上述7种数据类型之一来表示。
  • 只有7种数据类型似乎不足以表示全部数据。
  • 但ECMAScript的数据类型很灵活，一种数据类型可以当作多种数据类型来使用。

除以上数据类型之外还有一个数据类型

  • Bigint 整数类型

typeof 操作符

  • typeof 用于确定任意变量的数据类型；因typeof是操作符不是函数，所以不需要参数，但可以使用参数
  • 返回下列字符串之一
    • “undefined”: 未定义
    • “boolean” : 布尔值
    • “string” : 字符串
    • “number” : 数值
    • “object” : 对象（不是函数）或null
    • “function” : 函数
    • “symbol” : 符号
  • 特殊返回
    • ==typeof null 返回 object 因为 特殊值 null 被认为是一个空对象的引用==
  • 函数在 es 中被认为是对象，并不代表一种数据类型。但函数有自己的特殊属性，因此有必要通过 typeof 区分函数和其他对象

undefined 类型

  • undefined 类型只有一个值，即特殊值 undefined
  • 使用 var、let 声明变量但未初始化时，相当于给该变量赋值 undefined
  • 包含 undefined值的变量和未定义变量是有区别的
    • 对未声明的变量只能执行一个有用操作；即：调用 typeof 返回结果是该特殊值
    • 对未声明的变量调用 delete 在非严格模式下不报错；但没有实际用途
  • undefined 是一个假值，如果需要可以使用 if 语句进行检测，但需要明确检测的是 undefined 这个字面值，而不仅仅是假值

==注意：永远不给变量显式赋值undefined；字面值undefined主要用于比较；es3之前不存在，增加该特殊值的目的式为了正式明确空对象指针（null）和未初始化变量的区别==

null 类型

  • null 类型只有一个值，即特殊值 null
  • null 表示空对象指针；这也是 typeof null 返回 object 的原因
  • 初始化变量时推荐使用null初始化；只需要检查该变量的值就可以知道该变量是否重新赋值
  • undefined 由 null 派生的来，因此它们表面相等；即：undefined == null 的结果为 true
  • undefined 和 null 的用途不同；
    • 任何时候变量保存对象，但是没有对象可保存时使用null填充该变量
    • 这样就可以保持 null 是空对象指针的语义，并进一步区分 null 和 undefined
  • null 是一个假值，如果需要可以使用if语句进行检测，但需要明确检测的是null这个字面值，而不仅仅是假值

boolean 类型

  • boolean 类型有两个值
    • true
    • false
  • 布尔值字面量true、false区分大小写，因此True、False及其他大小混写形式是有效标识符但不是布尔值
  • Boolean(其他类型值) 可以将其他类型的值转换为布尔值

转换规则：

  • 理解转换规则非常重要，因为if等控制流语句会自动执行其他类型值转换为布尔值
  • 由于存在自动转换，理解控制流语句中使用的是什么变量就非常重要，错误使用对象而不是布尔值会明显改变应用程序的执行流
数据类型 转为true的值 转为false的值
Boolean true false
String 非空字符串 空字符串
Number 非零数值（包括无穷值） 0、NaN
Object 任意对象 null
Undefined N/A（不存在） undefined

Number 类型

  • Number 类型使用IEEE 754格式表示整数和浮点值（某些语言中称为双精度值）
  • 不同数值类型有对应的数值字面量格式；格式进制
    • 十进制
    • 八进制
    • 十六进制

1. 八进制

  • 以8为基数
  • 第一个数字必须是0，然后是相应的八进制数字0~7
  • 超出范围会忽略前缀0后续数字当成十进制数字
  • 严格模式下无效，js引擎抛出语法错误
    • 如果需要表示八进制值，通过前缀0o表示

2. 十进制

  • 正常书写即可

3. 十六进制

  • 前缀0x表示（区分大小写），然后是十六进制数字（09及AF）
  • 十六进制数字中字母大小写均可

八进制和十六进制格式创建的数值==在数学操作中被视为十进制数值==

==注：由于 js 保存数值的方式，可能存在正零和负零。正零和负零在任何情况下都被认为等同==

4. 浮点值（小数值）

  • 数值中必须包含小数点且小数点后必须至少有一位数字
  • 小数点前虽然不是必须有整数，但推荐添加
  • 浮点值是整数使用内存空间的两倍
  • 小数点后没有数字或小数点后跟着零，数值变为整数；
  • 对于非常大或非常小的数值，浮点值可以用科学计数法表示
    • 科学计数法：表示一个乘以10的给定次幂的数值
    • es中科学计数法格式：数值（整数或浮点数）+ 大/小写字母e + 乘以10的多少次幂
      let floatNum = 3.125e7 // 等于 31250000
/**
    本例中 floatNum = 31250000
    该表示法实际上相当于 以 3.125 为系数 乘以 10 的 7 次幂
*/
    • 科学计数法也用于表示非常小的数值。默认es会将小数点后至少包含6个零的浮点值转为科学计数法
    • 浮点值的精确度最高答17位，在算数计算中不如整数精确。由于这种微小的舍入错误导致难以测试特定的浮点值
      • 舍入错误是因为使用 IEEE 754 数值，使用该格式的语言均有此问题
        ```
        // 0.1 + 0.2 = 0.30000000000000004
        // 示例：
        if (a + b == 0.3) {
        console.log(“ You got 0.3.”)
        }
        /*
        检测两个数值的和是否等于0.3 如果是 0.1 和 0.2 这两个特定数值测试将失败
      */
      ```

      5. 值的范围

  • 因内存限制 es 不支持表示世界上所有数值；下列是特殊值
    • 最小数值： Number.MIN_VALUE ==> 5e-324
    • 最大数值： Number.MAX_VALUE ==> 1.7976931348623157e+308
    • 正无穷大（超出js可表示范围的正数）：Infinity
    • 负无穷大（超出js可表示范围的负数）：**-Infinity**
  • 计算返回 正负无穷则该值不能进行下一步计算，因无穷没有任何用于计算的数值表示
  • 确定给一个值是否是有限大（介于最大和最小数值之间）：isFinite()
    console.log(isFinite(Number.MAX_VALUE + Number.MAX_VALUE)) // false
  • 正负无穷大在计算时是有可能的，因此计算时有必要检测计算结果是否超出范围；两个属性的对应值下列列出
    • Number.NEGATIVE_INFINITY ==> -Infinity
    • Number.POSITIVE_INFINITY ==> Infinity

6. NaN

  • 特殊数值，意思是 “不是数值”（Not a Number）
  • 表示本来返回数值的操作失败，不是抛出错误
  • es中 0、+0、-0 相除返回 NaN；其他语言通常导致错误而终止代码执行
  • 分子是非0值，分母是有/无符号0 返回Infinity/-Infinity
  • 特殊属性
    • 任何涉及NaN的操作始终返回NaN
    • NaN不等于包括NaN在内的任何值
  • isNaN(any)
    • 该函数接收一个任意数据类型的参数并尝试将该参数转为数值，任何不能转换位数值的值都导致该函数返回true
    • 执行逻辑：
      • 该函数可用于测试对象，首先调用对象的valueof()
      • 然后确定返回值是否可以转为数值
      • 如果不能则调用toString()并测试返回值；这是es内置对象和操作符的执行方式

7. 数值转换

  • 将非数值转为数值的函数
    • Number()
      • 转型函数：用于任何数据类型转为Number 类型
      • 转换规则
        • boolean
          • true ==> 1
          • false ==> 0
        • Number ==> 直接返回
        • null ==> 0
        • undefined ==> NaN
        • String
          • 字符串包含字符，数字字符前带加、减号转为十进制数值，忽略前面的数字0
          • 字符串包含有效浮点值格式转为相应浮点值并忽略前面的数字0
          • 字符包含有效十六进制格式转为十六进制对应十进制整数
          • 空字符串且不包含字符转为数字0
          • 其他字符转为NaN
        • Object
          • 调用 valueof() 并按照上述规则转换返回值，返回值为NaN则调用 toString()再按照字符串规则转换
    • parseInt()
      • 解析字符串并返回有效数值的整数部分
      • 参数：
        • string：被解析的字符串 必填
        • radix：进制 选填
    • parseFloat()
      • 解析字符串并返回有效数值的第一次小数点位置
      • 忽略字符串开头的零
      • 只支持十进制

string 类型

  • 表示零或多个16位unicode字符序列化
  • 字符串可使用双引号、单引号、反引号标示
  • 字符串引号开头和结尾必须一致

1. 字符字面量

  • string类型包含字符字面量，用于表示非打印字符或其他用途字符
  • 这些字符字面量可出现再字符串中的任意位置且可作为单个字符被解释
  • 如下：
    字面量 含义
    \n 换行
    \t 制表
    \b 退格
    \r 回车
    \f 换页
    \ 反斜杠
    ' 单引号
    " 双引号
    ` 反引号
    \xnn 十六进制编码nn表示的字符（n是十六进制数字0~F）
    \unnnn 十六进制编码nnnn表示的unicode字符（n是十六进制数字0~F）
  • 字符串长度可通过length属性获取；若字符串中包含双字节符号，返回的值可能不是准确字符数
    let text = "this is the letter sigma"
console.log(text.length) // 24

2. 特点

  • es中字符串不可变（immutable），一旦创建值就不能改变了，若要修改变量中的字符串值，必须先销毁原始的字符串，然后将包含新值的另一个字符串保存到该变量

3. 转为字符串

  • toString()
    • 作用：返回当前值的字符串等价物
    • 拥有该方法的数据类型：数值、布尔值、对象、字符串值
    • 只有在数值调用该方法时接收一个底数参数（以什么底数输出数值的字符串表示）
    • 默认返回数值的十进制字符串表示；通过传入参数可获取二进制、八进制、十六进制、或其他任何有效基数的字符串表示
    • 默认（不传参数）的输出与传入参数10得到的结果一致
  • String()
    • 不确定值是否为null或undefined时使用
    • 始终返回表示相应类型值的字符串
    • 规则：
      • 值有toString()方法，则调用该方法（不传参数）并返回结果
      • 值为null 返回 “null”
      • 值为undefined 返回 “undefined”
  • 加号操作符给值加空字符串
    let num = 123
let str = num + ""
console.log(str, typeof str) // 123 string

4. 模板字面量

  • es6新增，与使用单双引号不同，模板字面量保留换行字符，可跨行定义字符串
  • 由于模板字面量会保持反引号内部空格，因此使用时需注意
  • 格式正确的模板字符穿可能看起来缩进不当

5. 字符串插值

  • 模板字面量最常用的一个特性时支持字符串插值；即可在一个连续定义插入一个或多个值
  • 技术角度模板字面量不是字符串，而是特殊的js句法表达式，求值之后得到的时字符串
  • 模板字面量在定义时立即求值并转换为字符串实例，任何插入的变量也会从其最接近的作用域中取值
  • 通过 ${}中js表达式实现
    let val = 5, exponent = 'second';
let interpolatedTemplateLiteral = `${ val } to the ${ exponent } power is ${ val * val}`
console.log(interpolatedTemplateLiteral) // 5 to the second power is 25
  • 所有插入值都使用toString()强制转为字符串且任何js表达式都可用于插值；嵌套的模板字符串无需转义
  • 插值表达式中可调用函数和方法
    function capitalize(word) {
  return `${word[0].toUpperCase()}${word.slice(1)}`
}
console.log(`${capitalize('hello')}, ${capitalize('world')}!`) // Hello, World!
  • 可插入自己之前的值
    let val = '';
function append() {
  val = `${val}abc`
  console.log(val)
}
append() //abc
append() // abcabc

6. 模板字面量标签函数

  • 模板字面量也支持定义标签函数（tag function），通过标签函数可自定义插值行为

  • 标签函数会接收被插值记号分隔后的模板和对每个表达式求值的结果

  • 标签函数本身时常规函数，通过前缀到模板字面量来应用自定义行为

    /**
* 标签函数接收到的参数一次是原始字符串数组和对每个表达式求值的结果
* 该函数的返回值是对模板字面量求值得到的字符串
*/
let a = 6, b = 9;
function simpleTag(strings, aValExp, bValExp, sumExp) {
  console.log(strings) // ["", " + ", " = ", ""]
  console.log(aValExp) // 6
  console.log(bValExp) // 9
  console.log(sumExp)  // 15
  return 'foobar'
}
let untaggedResult = `${ a } + ${ b } = ${ a + b }`,
    taggedResult = simpleTag`${ a } + ${ b } = ${ a + b }`;
console.log(untaggedResult) // "6 + 9 = 15"
console.log(taggedResult)   // "foobar"

// 表达式参数的数量可变，通常应使用剩余操作符（rest operator）将其收集到一个数组中
let a = 6, b = 9;
function simpleTag(strings, ...exps) {
  console.log(strings) // ["", " + ", " = ", ""]
  for (const exp of exps) console.log(exp);
  // 6
  // 9
  // 15
  return 'foobar'
}
let taggedResult = simpleTag`${ a } + ${ b } = ${ a + b }`
console.log(taggedResult) // "foobar"

  • 对有n个插值的模板字面量，传给标签函数的表达式参数个数始终是n

  • 传给标签函数第一个参数所包含的字符串个数始终是n+1

  • 如果想把该字符串和对表达式求值的结果拼接作为默认返回的字符串可参考下例

    let a = 6, b = 9;
function zipTag (strings, ...exps) {
  return strings[0] + exps.map((e, i) => `${ e }${ strings[i + 1] }`).join('')
}
let untaggedResult = `${ a } + ${ b } = ${ a + b }`,
    taggedResult = zipTag`${ a } + ${ b } = ${ a + b }`;
console.log(untaggedResult) // "6 + 9 = 15"
console.log(taggedResult)   // "6 + 9 = 15"

7. 原始字符串

  • 使用模板字面量也可直接获取原始模板字面量内容；如：换行符、unicode字符；而不是被转换后的字符表示，可使用默认的 String.raw 标签函数

    // unicode demo
// \u00A9 是版权符号
console.log(`\u00A9`) // ©
console.log(String.raw`\u00A9`) // \u00A9

// 换行符 示例
console.log(`first line\nesecond line`)
// first line
// nesecond line
console.log(String.raw`first line\nesecond line`) // "first line\nesecond line"

/**
 * 对实际的换行符来说是不行的
 * 它们不会被转换为转义序列的形式
*/
console.log(`first line
second line`)
// first line
// second line
console.log(String.raw`first line
second line`)
// first line
// second line
/**
 * 可通过标签函数的第一个参数
 * 字符串数组的.raw属性取得每个字符串的原始内容
*/
function printRaw (strs) {
  console.log('Actual characters')
  for (const str of strs) console.log(str)
  console.log('Escaped characters')
  for (const rawStr of strs.raw) console.log(rawStr)
}
printRaw`\u00A9${'and'}\n`
// Actual characters
// ©
// （换行符）
// Escaped characters
// \u00A9
// \n

object 类型

  • object是什么：es中object是一组数据和功能的集合
  • 创建方式：通过new操作符后跟对象类型的名称创建
    • 通过创建 Object类型的实例创建自己的对象，然后给对象添加属性和方法
      let obj = new Object()
// es只要求在给构造函数提供参数时使用括号
// 如果没有参数可省略括号（不推荐省略）
let obj = new Object
  • object本身实例并不重要重要的是要理解其相关概念
  • es中object也是派生其他对象的基类
  • object类型的所有属性和方法在派生对象上同样存在
  • 每个 object 实例都有如下属性和方法
    • constructor
      • 创建当前对象的函数，前面例子中该属性的值是 Object()
    • hasOwnProperty（propertyName）
      • 判断当前对象实例（非原型）上是否存在给定的属性
      • 需要检查的属性名必须是字符串（如：obj.hasOwnProperty(“name”)）或符号
    • isPrototypeOf(object)
      • 判断当前对象是否是另一个对象的原型
    • peopertyIsEnumerable（propertyName）
      • 判断给定的属性是否可以使用for-in语句枚举
      • 属性名必须是字符串
    • toLocaleString()
      • 返回对象的字符串表示
    • valueOf()
      • 返回都西昂对应字符串、数值、布尔值表示
      • 通常与 toString() 的返回值一致
    • 因在es中object是所有对象的基类，所以任何对象都有这些属性和方法
  • ==严格来讲ECMA-262中对象的行为不一定适合js中其他对象；如：浏览器中的BOM和DOM对象，都是由宿主环境定义和提供的宿主对象。宿主对现象不受ECMA-262约束，所以它们可能不继承 Object==

symbol 类型

  • es6新增数据类型
  • 符号是原始值且符号实例唯一不可变
  • 作用：
    • 确保对象属性使用唯一标识符，避免属性冲突
  • 描述：
    • 虽然和私有属性类似，但符号并不是为提供私有属性增加；尤其是因 Object Api提供方法可更方便发现符号属性
    • 符号用于创建唯一记号，用于非字符串形式的对象属性

1. 符号基本用法

  • 使用Symbol()函数初始化
  • 符号本身是原始类型 typeof 操作符返回 symbol
    console.log(typeof Symbol()) // "symbol"
  • 调用该函数可传入一个字符串参数作为对符号的描述（description）
    • 通过该字符串可调试代码
    • 该字符串参数与符号定义、标识完全无关
      console.log(Symbol() == Symbol('foo')) // false
  • 符号没有字面量语法
    • 按照规范，只要创建symbol()实例并用作对象新属性就可保证其不会覆盖已有对象属性，无论是符号属性或字符串属性
      let sym = Symbol()
let sym1 = Symbol('foo')
console.log(sym, sym1) // Symbol(), Symbol('foo')
  • symbol()函数不能与new关键字一起作为构造函数
    console.log(typeof new Boolean()) // "object"
console.log(typeof new String()) // "object"
console.log(typeof new Number()) // "object"
console.log(typeof new Symbol()) // TypeError: Symbol is not a constructor
  • 通过Object()函数可使用符号包装对象
    console.log(typeof Object(Symbol())) // "object"

2.1 使用全局符号注册表-symbol.for()

  • 运行时，不同部分需要共享和重用符号实例，可用一个字符串作为键，在全局符号注册表中创建并重用符号
    console.log(typeof Symbol.for('foo')) // symbol
  • 对每个字符串键都执行幂等操作，第一次调用时检查全局运行时注册表
    • 检索到不存在的对应符号时生成新符号实例并添加到注册表
    • 检索到对应符号则返回该符号实例
      let GloSym = Symbol.for('foo') // 创建新符号
let othGloSym = Symbol.for('foo') // 重用已有符号
console.log(GloSym === othGloSym) // true
  • 使用相同的符号描述在全局注册表中定义的符号和使用symbol()定义的符号不等同
    let locSym = Symbol('foo')
let gloSym = Symbol.for('foo')
console.log(locSym === gloSym) // false
  • 全局注册表中符号必须使用字符串键创建
    • 作为参数传给Symbol.for()的任何值都被转为字符串
    • 注册表中使用相同键被用作符号描述
      console.log(Symbol.for()) // Symbol(undefined)

2.2 使用全局符号注册表-Symbol.keyFor()

  • 用于查询全局注册表
  • 方法接收符号
  • 返回该全局符号对应的字符串键，查询的不是全局符号返回 undefined
    // 全局符号
let s = Symbol.for('foo')
console.log(Symbol.keyFor(s)) // foo
// 普通符号
let s1 = Symbol('bar')
console.log(Symbol.keyFor(s1)) // undefined
// 传给keyFor 的不是符号抛出 TypeError
Symbol.keyFor(123) // TypeError: 123 is not a symbol

3. 使用符号作为属性

  • 任何可使用字符串或数值作为属性的地方都可以使用符号
  • 包括对象字面量属性和 Object.defineProperty()/Object.defineProperties()定义的属性
  • 对象字面量只能在计算属性语法中使用符号作为属性
    let s1 = Symbol('foo'),
    s2 = Symbol('bar'),
    s3 = Symbol('baz'),
    s4 = Symbol('qux'),
    o = { [s1]: 'foo val' };
console.log('o', o) // o, {symbol(foo): "foo val"}
o[s1] = 'foo val';
console.log('o1', o) // o1, {symbol(foo): "foo val"}
Object.defineProperty(o, s2, { value: 'bar val' })
console.log('o', o) // o, {symbol(foo): "foo val"}, {symbol(bar): "bar val"}
Object.defineProperties(o, {
  [s3]: { value: 'baz value' },
  [s4]: { value: 'qux value' }
})
console.log(o)
  /**
   * o,
   * {symbol(foo): "foo val"},
   * {symbol(bar): "bar val"},
   * {symbol(baz): "baz val"},
   * {symbol(qux): "qux val"}
  */
  • 类似下列返回
    • Object.getOwnPropertyNames() 返回对象实例的常规属性数组；与下一种返回值相斥
    • Object.getOwnPropertySymbols() 返回对象实例的符号属性数组；与上一种返回值相斥
    • Object.getOwnPropertyDescriptors() 返回同时包含常规和符号属性描述符的对象
    • Reflect.ownKeys() 返回两种类型的键
      let s1 = Symbol('foo'),
  s2 = Symbol('bar'),
  o = {
    [s1]: 'foo val',
    [s2]: 'bar val',
    baz: 'baz val',
    qux: 'qux val'
  };
console.log(Object.getOwnPropertySymbols(o)) // [Symbol(foo), Symbol(bar)]
console.log(Object.getOwnPropertyNames(o)) // ["baz", "qux"]
console.log(Object.getOwnPropertyDescriptors(o))
/**
 * baz: {value: "baz val", writable: true, enumerable: true, configurable: true}
 * qux: {value: "qux val", writable: true, enumerable: true, configurable: true}
 * Symbol(bar): {value: "bar val", writable: true, enumerable: true, configurable: true}
 * Symbol(foo): {value: "foo val", writable: true, enumerable: true, configurable: true}
*/
console.log(Reflect.ownKeys(o)) // ["baz", "qux", Symbol(foo), Symbol(bar)]
  • 直接作为属性创建的符号不会丢失，如果不显示保存这些属性的引用必须遍历对象的所有符号属性才能找到对应属性键
    let o = { [Symbol('foo')]: 'foo val', [Symbol('bar')]: 'bar val' }
console.log(o) // {Symbol(foo): "foo val", Symbol(bar): "bar val"}
console.log(Object.getOwnPropertySymbols(o).find(symbol => symbol.toString().match(/bar/))) // Symbol(bar)

4. 常用内置符号

  • es6 引入一批常用内置符号（well-known symbol）
  • 作用：
    • 暴露语言内部行为
    • 开发者直接访问、重写、模型这些行为
    • 以Symbol工厂函数、字符串属性的形式存在
    • 最重要的用途之一就是重新定义而改变原生结构的行为；如：
      • for-of 循环会在相关对象上使用 Symbol.iterator属性
      • 可以通过自定义对象上重新定义该Symbol.iterator的值来改变 for-of 迭代对象时的行为
  • 内置符号是全局函数Symbol的普通字符串属性，指向一个符号的实例
  • 所有内置符号属性都是不可写、不可枚举、不可配置
  • es规范，引用符号在规范中的名称，前缀为@@；如：@@iterator指的是 Symbol.iterator

5. Symbol.asyncIterator

  • 根据es 2018规范定义，该符号作为一个属性表示
    • “一个方法，返回对象默认的AsyncIterator。由for-await-of语句使用”
    • 该符号表示实现异步迭代器API的函数
  • for-await-of循环会利用该函数执行异步迭代操作
    • 循环时调用以 Symbol.asyncIterator为键的函数
    • 期望该函数返回一个实现迭代器API的对象
    • 大多时返回的对象是实现该API的AsyncIterator
      ``` js
      class Foo { async [Symbol.asyncIterator] () {} }
      let f = new Foo()
      console.log(fSymbol.asyncIterator) // AsyncGenerator {}
      /      *
  • proto: AsyncGenerator
  • [[GeneratorLocation]]: VM375:1 ==> class Foo 对象所在行数
  • [[GeneratorState]]: “suspended”
  • [[GeneratorFunction]]: ƒ async *Symbol.asyncIterator
  • [[GeneratorReceiver]]: Foo
  • [[Scopes]]: Scopes[3]
  • /
    ```
  • 技术上由Symbol.asyncIterator函数生成的对象
    • 应通过其 next() 方法陆续返回 Promise 实例
    • 可通过显示调用 next() 方法返回
    • 可隐式通过异步生成器函数返回
      class Emitter {

  this.max = max
  this.asyncIdx = 0
}
async *[Symbol.asyncIterator] () {

    yield new Promise((resolve) => resolve(this.asyncIdx++))

}
}

}

6. Symbol.hasInstance

  • 根据es规范，该符号作为一个属性表示
    • 一个方法，该方法决定一个构造器对象是否认可一个对象是它的实例
    • 由 instanceof 操作符使用。该操作符可确定一个对象实例的原型链上是否有原型
      // 使用场景
function Foo () {}
console.log(new Foo() instanceof Foo) // true
class Bar {}
console.log(new Bar() instanceof Bar) // true
  • es6 中 instanceof操作符使用Symbol.hasInstance 函数确定关系
    • 以该方法为键的函数会执行相同操作，只是操作数对调
      function Foo () {}
console.log(Foo[Symbol.hasInstance](new Foo())) // true
class Bar {}
console.log(Bar[Symbol.hasInstance](new Bar())) // true
  • 该属性在Function原型上定义
  • 默认在所有函数和类上都可以调用
  • 可在继承的类上通过静态方法重定义该函数
    • 因 instanceof 操作符会在原型上查找该属性定义
    • 等同在原型链上查找其他属性一样
      class Bar {}
class Baz extends Bar {
static [Symbol.hasInstance] () { return false }
}
let b = new Baz()
console.log(Bar[Symbol.hasInstance](b)) // true
console.log(b instanceof Bar) // true
console.log(Baz[Symbol.hasInstance](b)) // false
console.log(b instanceof Baz) // false

7. Symbol.isConcatSpreadable

  • 根据es规范该符号作为一个属性表示为boolean值
    • true：则对象应该用Array.prototype.concat()打平其数组元素
      • concat()方法根据接收的对象类型选择如何将一个类数组对象拼接成数组实例
      • 覆盖Symbol.isConcatSpreadable的值可修改该行为
  • 数组对象默认被打平到已有数组，false或假值：导致整个对象被追加到数组末尾
  • 类数组对象默认追加到数组末尾，true或真值：导致该类数组对象被打平到数组实例
  • 其他非类数组对象的对象在 Symbol.isConcatSpreadable 被设置为true时被忽略
    ``` js
    let initial = [‘foo’]
    let array = [‘bar’]
    console.log(array[Symbol.isConcatSpreadable]) // undefined
    console.log(initial.concat(array)) // [“foo”, “bar”]
    array[Symbol.isConcatSpreadable] = false
    console.log(initial.concat(array))
    // [“foo”, Array(1)] ==> [“foo”, [“bar”, Symbol(Symbol.isConcatSpreadable): false]]

let arrayLikeObject = { length: 1, 0: ‘baz’ }
console.log(arrayLikeObject[Symbol.isConcatSpreadable]) // undefined
console.log(initial.concat(arrayLikeObject))
// [“foo”, {…}] ==> [“foo”, {0:”baz”, length:1, Symbol(Symbol.isConcatSpreadable): true}]
arrayLikeObject[Symbol.isConcatSpreadable] = true
console.log(initial.concat(arrayLikeObject)) // [“foo”, “baz”]

let otherObject = new Set().add(‘qux’)
console.log(otherObject[Symbol.isConcatSpreadable]) // undefined
console.log(initial.concat(otherObject))
// [“foo”, Set(1)] ==> [0: “foo”, 1: Set(1) {“qux”}]
otherObject[Symbol.isConcatSpreadable] = true
console.log(initial.concat(otherObject)) // [“foo”]


### 8. Symbol.iterator
- 根据es规范该符号作为属性表示一个方法
  - 该方法返回对象默认迭代器，由for-of语句使用；即：该符号表示实现迭代器API的函数
  - for-of循环这样的语言结构利用该函数执行迭代器操作
    - 循环时调用以Symbol.iterator为键的函数
    - 默认该函数返回一个实现迭代器API的对象
    - 返回的对象是实现该API的 Generator

    ``` js
    class Foo {
      *[Symbol.iterator] () {}
    }
    let f = new Foo()
    console.log(f[Symbol.iterator]()) // Generator {<suspended>} ==>
    /***
    __proto__: Generator
    [[GeneratorLocation]]: VM269:2
    [[GeneratorState]]: "suspended"
    [[GeneratorFunction]]: ƒ* [Symbol.iterator]()
    [[GeneratorReceiver]]: Foo
    [[Scopes]]: Scopes[3]
    */
    ```
- 技术上这个由 Symbol.iterator 函数生成的对象应通过其next()方法陆续返回值
  - 可通过显示调用next()方法返回
  - 可隐式通过生成器函数返回
  ``` js
  class Emitter {
    constructor (max) {
      this.max = max
      this.idx = 0
    }
    *[Symbol.iterator] () {
      while (this.idx < this.max) yield this.idx++
    }
  }
  function count () {
    let emitter =  new Emitter(5)
    for (const x of emitter) console.log(x)
  }
  count()
  /**
    0
    1
    2
    3
    4
  */
  ```

### 9. Symbol.match
- 根据es规范该符号作为属性表示正则表达式方法
  - 该方法用正则表达式匹配字符串
  - 由String.prototype.match()方法使用
- 该方法使用Symbol.match为键的函数对正则表达式求值
  - 所有正则表达式实例默认该String方法的有效参数
    - 正则表达式原型上默认有该函数定义
  ``` js
  console.log(RegExp.prototype[Symbol.match])
    // [Symbol.match]() {[native code]}
  console.log('foobar'.match(/bar/))
    // ["bar", index: 3, input: "foobar", groups: undefined]
  ```
- 该方法传入非正则表达式值会将该值转为RegExp对象；如果需要改变此行为
  - 可通过方法直接使用参数，可重定义Symbol.match函数取代默认对正则表达式求值的行为
    - 从而使match()方法使用非正则表达式实例
- Symbol.match函数接收一个参数
  - 调用match()方法的字符串实例
- 返回值无限制
  ``` js
  class FooMatcher {
    static [Symbol.match] (target) {
      return target.includes('foo')
    }
  }
  console.log('foobar'.match(FooMatcher)) // true
  console.log('barbaz'.match(FooMatcher)) // false

  class StringMatcher {
    constructor (str) {
      this.str = str
    }
    [Symbol.match] (target) {
      return target.includes(this.str)
    }
  }
  console.log('foobar'.match(new StringMatcher('foo'))) // true
  console.log('barbaz'.match(new StringMatcher('qux'))) // false
  ```

### 10. Symbol.replace
- 根据es规范该符号作为属性表示正则表达式方法
  - 该方法替换字符串中匹配的子串
  - 由String.prototype.replace()方法使用
- 该方法使用Symbol.replace为键的函数对正则表达式求值
- 所有正则表达式实例默认该String方法的有效参数
  - 正则表达式原型上默认有该函数定义
  ``` js
  console.log(RegExp.prototype[Symbol.replace])
    // f [Symbol.replace]() {[native code]}
  console.log('foobar'.replace(/bar/, 'qux')) // 'fooqux'
  ```
  - 该方法传入非正则表达式值该值会转为RegExp对象；如果需要改变此行为
    - 方法直接使用参数，可重定义Symbol.replace函数取代默认对正则表达式的求值行为
      - 从而使 replace()方法使用非正则表达式实例
- Symbol.replace函数接收两个参数
  - 调用replace()方法的字符串实例
  - 替换字符串
- 返回值无限制
  ``` js
  class FooReplacer {
    static [Symbol.replace] (target, replacement) {
      return target.split('foo').join(replacement)
    }
  }
  console.log('barfoobaz'.replace(FooReplacer, 'qux')) // "barquxbaz"

  class StringReplacer {
    constructor (str) {
      this.str = str
    }
    [Symbol.replace] (target, replacement) {
      return target.split(this.str).join(replacement)
    }
  }
  console.log('barfoobaz'.replace(new StringReplacer('foo'), 'qux')) // "barquxbaz"
  ```

### 11. Symbol.search
- 根据es规范，该符号作为属性表示正则表达式方法
  - 该方法返回字符串中匹配正则表达式的索引
  - 由Symbol.prototype.search()方法使用
- 该方法使用以Symbol.search为键的函数对正则表达式求值
- 所有正则表达式实例默认是该String的有效参数

``` js
console.log(RegExp.prototype[Symbol.search])

  • Symbol.search函数接收一个参数

12. Symbol.species

  • 根据es规范，该符号作为属性表示一个函数值
    • 该函数作为创建派生对象的构造函数
  • 该属性常用于内置类型
    • 用于对内置类型实例方法的返回值暴露实例化派生对象的方法
  • 用于Symbol.species定义静态获取器（getter）方法
  • 可覆盖新创建实例的原型定义
    ``` js
    class Bar extends Array {}
    class Baz extends Array {
    static get [Symbol.species] () {
    return Array
    }
    }
    let bar = new Bar()
    console.log(bar instanceof Array) // true
    console.log(bar instanceof Bar) // true
    bar = bar.concat(‘bar’)
    console.log(bar instanceof Array) // true
    console.log(bar instanceof Bar) // true

let baz = new Baz()
console.log(baz instanceof Array) // true
console.log(baz instanceof Baz) // true
baz = baz.concat(‘baz’)
console.log(baz instanceof Array) // true
console.log(baz instanceof Baz) // false


### 13. Symbol.split
- 根据es规范，该符号作为属性表示正则表达式方法
  - 该方法在匹配正则表达式的索引位置拆分字符串
  - 由String.prototype.split()方法使用
- 该方法使用以Symbol.split为键的函数对正则表达式求值
- 所有正则表达式实例默认是这个String方法的有效参数
  - 正则表达式原型上默认由该函数定义
  ```js
  console.log(RegExp.prototype[Symbol.split])
    // f [Symbol.split] () {[native code]}
  console.log('foobarbaz'.split(/bar/)) // ["foo", "baz"]
  ```
- 给该方法传入非正则表达式值，值转为RegExp对象；若需要改变此行为
  - 方法直接使用参数，可重定义Symbol.split函数取代默认对正则表达式求值行为
  - 从而让split()方法使用非正则表达式实例
- Symbol.split 函数接收一个参数
  - 调用match()方法的字符串实例
- 返回值无限制
``` js
class FooSpliter {
  static [Symbol.split] (target) {
    return target.split('foo')
  }
}
console.log('barfoobaz'.split(FooSpliter)) // ["bar", "baz"]

class StringSplitter {
  constructor (str) {
    this.str = str
  }
  [Symbol.split] (target) {
    return target.split(this.str)
  }
}
console.log('barfoobaz'.split(new StringSplitter('foo'))) // ["bar", "baz"]

14. Symbol.toPrimitive

  • 根据es规范，该符号作为属性表示一个方法
    • 该方法将对象转为相应原始值由ToPrimitive抽象操作使用
  • 很多内置操作都会尝试强制将对象转为原始值
    • 包括字符串、数值、未指定的原始类型
  • 对一个自定义对象实例，通过该实例的Symbol.toPrimitive属性上定义一个可改变默认行为的函数
    • 根据提供给该函数的参数（string、number、default）可控制返回的原始值
      ``` js
      class Foo {}
      let foo = new Foo()
      console.log(3 + foo) // “3[object Object]”
      console.log(3 - foo) // “NaN”
      console.log(String(foo)) // “[object Object]”

class Bar {
constructor () {
this[Symbol.toPrimitive] = function (hint) {
switch (hint) {
case ‘number’:
return 3
case ‘string’:
return ‘string bar’
case ‘default’:
default:
return ‘default bar’
}
}
}
}
let bar = new Bar()
console.log(3 + bar) // “3 default bar”
console.log(3 - bar) // 0
console.log(String(bar)) // “string bar”


### 15. Symbol.toStringTag
- 根据es规范，该符号作为一个属性表示一个字符串
  - 该字符串用于创建对象的默认字符串描述
  - 由内置方法 Object.prototype.toString使用
- 通过toString方法获取对象标识时
  - 检索由Symbol.toString指定的实例标识符,默认为 "Object"
  - 内置类型指定该值，但自定义类实例需明确定义
  ``` js
  let s = new Set()
  console.log(s) // Set(0) {}
  /**
  [[Entries]]
  No properties
  size: 0
  __proto__: Set
  */
  console.log(s.toString()) // [object Object]
  console.log(s[Symbol.toStringTag]) // Set

  class Foo {}
  let foo = new Foo()
  console.log(foo) // Foo {}
  console.log(foo.toString()) // [object Object]
  console.log(foo[Symbol.toStringTag]) // undefined

  class Bar {
    constructor () {
      this[Symbol.toStringTag] = 'Bar'
    }
  }
  let bar = new Bar()
  console.log(bar) // Bar {} ==> Bar {Symbol(Symbol.toStringTag): "Bar"}
  console.log(bar.toString()) // [object Bar]
  console.log(bar[Symbol.toStringTag]) // Bar
  ```

### 16. Symbol.unscopables
- 根据es规范，该符号作为属性表示对象
  - 该对象所有的以及继承的属性，都会从关联对象的with环境中绑定排除
- 设置该符号并使其映射对应属性的键值为true即可阻止该属性出现在with环境绑定中
  ``` js
  let o = { foo: 'bar' }
  with (o) { console.log(foo) } // bar
  o[Symbol.unscopables] = { foo: true }
  with (o) { console.log(foo) } // ReferenceError: foo is not defined
  ```
- ==**注意：不推荐使用with，因此也不推荐使用该方法 **==

### symbol类型作用
- 确保对象属性使用唯一标识符，避免属性冲突

### 描述
- 虽然和私有属性类似，但符号并不是为提供私有属性增加；尤其是因Object Api提供方法可更方便发现符号属性
- 符号用于创建唯一记号，用于非字符串形式的对象属性
- 没有字面量语法
- 不能和 new 关键字一起使用成为构造函数

### 方法
- Symbol.for() ==> 重用已有符号
- Symbol.keyfor() ==> 查询全局注册表
- Symbol.asyncIterator ==> 返回异步迭代器对象
- Symbol.iterator ==> 实现迭代器API的函数
- Symbol.hasInstance ==> 确定对象实例的原型链上是否有原型
- Symbol.isConcatSpreadable ==> 类数组对象拼接为数组实例
- Symbol.match ==> 正则表达式匹配字符串
- Symbol.replace ==> 正则表达式 替换字符串中匹配的子串
- Symbol.search ==> 返回字符串中匹配正则表达式的索引
- Symbol.species ==> 作为创建派生对象的构造函数
- Symbol.split ==> 在匹配正则表达式的索引位置拆分字符串
- Symbol.toPrimitive ==> 将对象转为相应的原始值
- Symbol.toStringTag ==> 创建对象的默认字符串描述
- Symbol.unscopables ==> 该对象所有的及继承的属性都从关联对象的with环境中绑定排除（不推荐）
