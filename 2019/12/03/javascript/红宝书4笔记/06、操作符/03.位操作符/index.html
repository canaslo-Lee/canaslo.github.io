03.位操作符

阅读量: 静思 2019-12-03 10:18:12
Categories： javascript Tags： js基础

位操作符

  • 用于数值底层操作，即：操作内存中表示数据的比特（位）
  • es中所有数值都以 IEEE 754 64位格式存储
  • 位操作并不直接应用到64位表示，而是先将值转为32位整数再进行位操作，之后将结果转为64位
  • 对开发者好像只有32位整数，64位置整数存储格式是不可见的，因此只需要考虑32位整数即可
  • 有符号整数使用32位的前31位表示整数值，第32位表示数值符号；如： 0表示正；1表示负
    • 这一位称为符号位(sign bit)
    • 它的值决定数值其余部分的格式
    • 正值以真正的二进制格式存储；即：31位中每一位都代表2的幂
      • 第一位（称为第0位）表示2⁰，第二位表示2¹，依次类推。若第一个位是空的则以0填充，相当于忽略不计
      • 如：数值18的二进制为00000000000000000000000000010010或10010，后者是用到5个有效位决定实际值
            1          0          0          1          0
(2⁴ * 1) + (2³ * 0) + (2² * 0) + (2¹ * 1) + (2⁰ * 0)
    16   +     0    +     0    +     2    +     0
                    18
  • 负值以一种称为二补数（补码）的二进制编码存储，一个数值的二补数通过下列步骤计算
    1. 确定绝对值的二进制表示；如：-18，先确定18的二进制表示

      0000 0000 0000 0000 0000 0000 0001 0010

    2. 找到数值的一补数（反码）；即：每个0变为1，每个1变为0

      1111 1111 1111 1111 1111 1111 1110 1101

    3. 给结果加1

      1111 1111 1111 1111 1111 1111 1110 1110

    4. -18 的二进制表示为11111111111111111111111111101110
    • 注意：处理有符号整数时无法访问第31位
    • 将负值输出位二进制字符串时，会得到一个前面加减号的绝对值
      let num = -18
console.log(num.toString(2))
  // "-10010"
  • 注意：
    • 默认es中所有整数都表示位有符号数
    • 确实存在无符号整数
    • 对无符号整数的第32位不表示符号，因为只有正值
    • 无符号整数比有符号整数的范围更大，因为符号位被用于表示数值
  • 对es数值应用位操作符时，后台发生转换
    • 64位转为32位数值
    • 然后执行位操作
    • 最后将结果从32位转为64位存储
    • 此操作与其他语言类似，导致出现一奇特副作用
      • 特殊值NaN、Infinity再位操作符中被当成0处理
  • 位操作符应用到非数值
    • 首先使用Number()函数将值转为数值（自动处理）
    • 再应用位操作
    • 最终结果是数值

1. 按位非

  • 作用
    • 返回数值的一补数
  • 描述：
    • 波浪符（~）表示
    • 是es中为数不多的几个二进制数学操作符之一
      console.log(~25)
// 25 = 二进制 00000000000000000000000000011001
// 结果 -26 = 符号 + 二进制 11111111111111111111111111100110
    • 按位非的最终效果是对数值取反并减1
      console.log(~25 - 1)
  // "-26"
    • 虽上述两种方法返回结果一致，但位操作的速度更快，因为位操作是在数值底层表示上完成

2. 按位与

  • 作用

    • 将两个数的每一个位对齐，然后基于真值表中的规则对每一位执行相应的与操作

  • 描述：

    • 和号（&）表示
    • 两个操作数
    • 两个位都是1时返回1，任何一位是0返回0

  • 示例：对数值25和3求与操作

    console.log(25 & 3)
  // 1

// 详解：
// 25  = 0000 0000 0000 0000 0000 0000 0001 1001
//  3  = 0000 0000 0000 0000 0000 0000 0000 0011
// ------------------------------------------
// AND = 0000 0000 0000 0000 0000 0000 0000 0001
// AND = 1

3. 按位或

  • 作用：

    • 将两个数的每一个位对齐，然后基于真值表中的规则对每一位执行相应的或操作

  • 描述：

    • 管道符（|）表示
    • 两个操作数
    • 两个位都是0时返回0，任何一位是1返回1

  • 示例：对数值25和3执行按位或操作

    console.log(25 | 3)
  // 27

// 详解：
// 25 = 0000 0000 0000 0000 0000 0000 0001 1001
//  3 = 0000 0000 0000 0000 0000 0000 0000 0011
// ------------------------------------------
// OR = 0000 0000 0000 0000 0000 0000 0001 1011
// 二进制码 11011 = 27
// OR = 27

4. 按位异或

  • 作用

    • 将两个数的每一个位对齐，然后基于真值表中的规则对每一位执行相应的异或操作

  • 描述：

    • 脱字符（^）表示
    • 两个操作数
    • 两位都是1或0时返回0，任何一位是1返回1
    • 比对同样两个值执行按位或操作得到的记过小1

  • 示例：对数值25和3执行按位异或操作

    console.log(25 ^ 3)
  // 26

// 详解：
// 25  = 0000 0000 0000 0000 0000 0000 0001 1001
//  3  = 0000 0000 0000 0000 0000 0000 0000 0011
// ------------------------------------------
// XOR = 0000 0000 0000 0000 0000 0000 0001 1010
// 二进制码 11010 = 26
// XOR = 26

5. 左移

  • 描述：
    • 两个小于号（<<）表示
    • 按照指定位数将数值的所有位向左移动
    • 数值右端因左移导致的空位以0填充，使结果是完整的32位
    • 保留其操作数值的符号；如：-2左移5位得到的是-64
  • 示例：数值2（二进制10）向左移5位，得到64（二进制1000000）
    console.log(2 << 5)
  // 64 二进制 1000000
// 有符号左移
console.log(-2 << 5)
  // -64 符号 + 二进制数值

6. 有符号右移

  • 描述：
    • 两个大于号（>>）表示
    • 将数值所有32位向右移
    • 保留其操作数值的符号
    • 有符号右移是左移的逆运算
    • 右移后在符号位后存在空位，es通过符号位的值填充空位，以得到完整数值
  • 示例：64右移5位，得到2
    console.log(64 >> 5)
  // 2

7. 无符号右移

  • 描述：
    • 三个大于号（>>>）表示
    • 将数值所有32位右移
    • 正数：与有符号右移结果一致
    • 负数：将负数的二进制表示当成正数的二进制表示处理
      • 因负数时其绝对值的二补数，所以右移后结果变得非常大
  • 示例：-64右移5位，得到134217726
    console.log(-64 >>> 5)
// -64 二进制：1111 1111 1111 1111 1111 1111 1100 0000
// 正值结果是： 42 9496 7232
// 右移5位后：0000 0111 1111 1111 1111 1111 1111 1110
// 结果 十进制：134217726
