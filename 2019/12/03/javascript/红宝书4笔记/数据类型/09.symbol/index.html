09.symbol

阅读量: 静思 2019-12-03 10:18:12
Categories： javascript Tags： js基础

描述

  • es6新增数据类型
  • 符号是原始值且符号实例唯一不可变
  • 作用：
    • 确保对象属性使用唯一标识符，避免属性冲突
  • 描述：
    • 虽然和私有属性类似，但符号并不是为提供私有属性增加；尤其是因 Object Api提供方法可更方便发现符号属性
    • 符号用于创建唯一记号，用于非字符串形式的对象属性

1. 符号基本用法

  • 使用Symbol()函数初始化
  • 符号本身是原始类型 typeof 操作符返回 symbol
    console.log(typeof Symbol())
  // "symbol"
  • 调用该函数可传入一个字符串参数作为对符号的描述（description）
    • 通过该字符串可调试代码
    • 该字符串参数与符号定义、标识完全无关
      console.log(Symbol() == Symbol('foo'))
  // false
  • 符号没有字面量语法
    • 按照规范，只要创建symbol()实例并用作对象新属性就可保证其不会覆盖已有对象属性，无论是符号属性或字符串属性
      let sym = Symbol()
let sym1 = Symbol('foo')
console.log(sym, sym1)
  // Symbol(), Symbol('foo')
  • symbol()函数不能与new关键字一起作为构造函数
    console.log(typeof new Boolean())
  // "object"
console.log(typeof new String())
  // "object"
console.log(typeof new Number())
  // "object"
console.log(typeof new Symbol())
  // TypeError: Symbol is not a constructor
  • 通过Object()函数可使用符号包装对象
    console.log(typeof Object(Symbol()))
  // "object"

2. 使用全局符号注册表

2.1 symbol.for()

  • 运行时，不同部分需要共享和重用符号实例，可用一个字符串作为键，在全局符号注册表中创建并重用符号
    console.log(typeof Symbol.for('foo'))
  // symbol
  • 对每个字符串键都执行幂等操作，第一次调用时检查全局运行时注册表
    • 检索到不存在的对应符号时生成新符号实例并添加到注册表
    • 检索到对应符号则返回该符号实例
      let GloSym = Symbol.for('foo')
  // 创建新符号
let othGloSym = Symbol.for('foo')
  // 重用已有符号
console.log(GloSym === othGloSym)
  // true
  • 使用相同的符号描述在全局注册表中定义的符号和使用symbol()定义的符号不等同
    let locSym = Symbol('foo')
let gloSym = Symbol.for('foo')
console.log(locSym === gloSym)
  // false
  • 全局注册表中符号必须使用字符串键创建
    • 作为参数传给Symbol.for()的任何值都被转为字符串
    • 注册表中使用相同键被用作符号描述
      console.log(Symbol.for())
// Symbol(undefined)

2.2 Symbol.keyFor()

  • 用于查询全局注册表
  • 方法接收符号
  • 返回该全局符号对应的字符串键，查询的不是全局符号返回 undefined
    // 全局符号
let s = Symbol.for('foo')
console.log(Symbol.keyFor(s))
// foo
// 普通符号
let s1 = Symbol('bar')
console.log(Symbol.keyFor(s1))
// undefined
// 传给keyFor 的不是符号抛出 TypeError
Symbol.keyFor(123)
// TypeError: 123 is not a symbol

3. 使用符号作为属性

  • 任何可使用字符串或数值作为属性的地方都可以使用符号
  • 包括对象字面量属性和 Object.defineProperty()/Object.defineProperties()定义的属性
  • 对象字面量只能在计算属性语法中使用符号作为属性
    let s1 = Symbol('foo'),
    s2 = Symbol('bar'),
    s3 = Symbol('baz'),
    s4 = Symbol('qux'),
    o = { [s1]: 'foo val' };
console.log('o', o)
  // o, {symbol(foo): "foo val"}
o[s1] = 'foo val';
console.log('o1', o)
  // o1, {symbol(foo): "foo val"}
Object.defineProperty(o, s2, { value: 'bar val' })
console.log('o', o)
  // o, {symbol(foo): "foo val"}, {symbol(bar): "bar val"}
Object.defineProperties(o, {
  [s3]: { value: 'baz value' },
  [s4]: { value: 'qux value' }
})
console.log(o)
  /**
   * o,
   * {symbol(foo): "foo val"},
   * {symbol(bar): "bar val"},
   * {symbol(baz): "baz val"},
   * {symbol(qux): "qux val"}
  */
  • 类似下列返回
    • Object.getOwnPropertyNames() 返回对象实例的常规属性数组；与下一种返回值相斥
    • Object.getOwnPropertySymbols() 返回对象实例的符号属性数组；与上一种返回值相斥
    • Object.getOwnPropertyDescriptors() 返回同时包含常规和符号属性描述符的对象
    • Reflect.ownKeys() 返回两种类型的键
      let s1 = Symbol('foo'),
  s2 = Symbol('bar'),
  o = {
    [s1]: 'foo val',
    [s2]: 'bar val',
    baz: 'baz val',
    qux: 'qux val'
  };
console.log(Object.getOwnPropertySymbols(o))
// [Symbol(foo), Symbol(bar)]
console.log(Object.getOwnPropertyNames(o))
// ["baz", "qux"]
console.log(Object.getOwnPropertyDescriptors(o))
/**
 * baz: {value: "baz val", writable: true, enumerable: true, configurable: true}
 * qux: {value: "qux val", writable: true, enumerable: true, configurable: true}
 * Symbol(bar): {value: "bar val", writable: true, enumerable: true, configurable: true}
 * Symbol(foo): {value: "foo val", writable: true, enumerable: true, configurable: true}
*/
console.log(Reflect.ownKeys(o))
// ["baz", "qux", Symbol(foo), Symbol(bar)]
  • 直接作为属性创建的符号不会丢失，如果不显示保存这些属性的引用必须遍历对象的所有符号属性才能找到对应属性键
    let o = {
  [Symbol('foo')]: 'foo val',
  [Symbol('bar')]: 'bar val'
}
console.log(o)
  // {Symbol(foo): "foo val", Symbol(bar): "bar val"}
console.log(Object.getOwnPropertySymbols(o).find(
  symbol => symbol.toString().match(/bar/)
))
  // Symbol(bar)

4. 常用内置符号

  • es6 引入一批常用内置符号（well-known symbol）
  • 作用：
    • 暴露语言内部行为
    • 开发者直接访问、重写、模型这些行为
    • 以Symbol工厂函数、字符串属性的形式存在
    • 最重要的用途之一就是重新定义而改变原生结构的行为；如：
      • for-of 循环会在相关对象上使用 Symbol.iterator属性
      • 可以通过自定义对象上重新定义该Symbol.iterator的值来改变 for-of 迭代对象时的行为
  • 内置符号是全局函数Symbol的普通字符串属性，指向一个符号的实例
  • 所有内置符号属性都是不可写、不可枚举、不可配置
  • es规范，引用符号在规范中的名称，前缀为@@；如：@@iterator指的是 Symbol.iterator

5. Symbol.asyncIterator

  • 根据es 2018规范定义，该符号作为一个属性表示
    • “一个方法，返回对象默认的AsyncIterator。由for-await-of语句使用”
    • 该符号表示实现异步迭代器API的函数
  • for-await-of循环会利用该函数执行异步迭代操作
    • 循环时调用以 Symbol.asyncIterator为键的函数
    • 期望该函数返回一个实现迭代器API的对象
    • 大多时返回的对象是实现该API的AsyncIterator
      class Foo { async *[Symbol.asyncIterator] () {} }
let f = new Foo()
console.log(f[Symbol.asyncIterator]())
// AsyncGenerator {<suspended>}
/**
 * __proto__: AsyncGenerator
* [[GeneratorLocation]]: VM375:1  ==> class Foo 对象所在行数
* [[GeneratorState]]: "suspended"
* [[GeneratorFunction]]: ƒ async *[Symbol.asyncIterator]()
* [[GeneratorReceiver]]: Foo
* [[Scopes]]: Scopes[3]
*/
  • 技术上由Symbol.asyncIterator函数生成的对象
    • 应通过其 next() 方法陆续返回 Promise 实例
    • 可通过显示调用 next() 方法返回
    • 可隐式通过异步生成器函数返回
      class Emitter {
constructor (max) {
  this.max = max
  this.asyncIdx = 0
}
async *[Symbol.asyncIterator] () {
  while (this.asyncIdx < this.max) {
    yield new Promise((resolve) => resolve(this.asyncIdx++))
  }
}
}
async function asyncCount () {
let emitter = new Emitter(5)
for await(const x of emitter) console.log(x)
}
asyncCount()
/**
 * 0
 * 1
 * 2
 * 3
 * 4
 * Promise{<fulfilled>: undefined}
    __proto__: Promise
    [[PromiseState]]: "fulfilled"
    [[PromiseResult]]: undefined
 */

6. Symbol.hasInstance

  • 根据es规范，该符号作为一个属性表示
    • 一个方法，该方法决定一个构造器对象是否认可一个对象是它的实例
    • 由 instanceof 操作符使用。该操作符可确定一个对象实例的原型链上是否有原型
      // 使用场景
function Foo () {}
console.log(new Foo() instanceof Foo)
  // true
class Bar {}
console.log(new Bar() instanceof Bar)
  // true
  • es6 中 instanceof操作符使用Symbol.hasInstance 函数确定关系
    • 以该方法为键的函数会执行相同操作，只是操作数对调
      function Foo () {}
console.log(Foo[Symbol.hasInstance](new Foo()))
  // true
class Bar {}
console.log(Bar[Symbol.hasInstance](new Bar()))
  // true
  • 该属性在Function原型上定义
  • 默认在所有函数和类上都可以调用
  • 可在继承的类上通过静态方法重定义该函数
    • 因 instanceof 操作符会在原型上查找该属性定义
    • 等同在原型链上查找其他属性一样
      class Bar {}
class Baz extends Bar {
static [Symbol.hasInstance] () { return false }
}
let b = new Baz()
console.log(Bar[Symbol.hasInstance](b))
// true
console.log(b instanceof Bar)
// true
console.log(Baz[Symbol.hasInstance](b))
// false
console.log(b instanceof Baz)
// false

7. Symbol.isConcatSpreadable

  • 根据es规范该符号作为一个属性表示为boolean值

    • true：则对象应该用Array.prototype.concat()打平其数组元素
      • concat()方法根据接收的对象类型选择如何将一个类数组对象拼接成数组实例
      • 覆盖Symbol.isConcatSpreadable的值可修改该行为

  • 数组对象默认被打平到已有数组，false或假值：导致整个对象被追加到数组末尾

  • 类数组对象默认追加到数组末尾，true或真值：导致该类数组对象被打平到数组实例

  • 其他非类数组对象的对象在 Symbol.isConcatSpreadable 被设置为true时被忽略

    let  initial = ['foo']
let array = ['bar']
console.log(array[Symbol.isConcatSpreadable])
  // undefined
console.log(initial.concat(array))
  // ["foo", "bar"]
array[Symbol.isConcatSpreadable] = false
console.log(initial.concat(array))
  /*
  [
    "foo",
    Array(1)
  ] ==> [
      "foo",
      [
        "bar",
        Symbol(Symbol.isConcatSpreadable): false
      ]
  ] */

let arrayLikeObject = { length: 1, 0: 'baz' }
console.log(arrayLikeObject[Symbol.isConcatSpreadable])
  // undefined
console.log(initial.concat(arrayLikeObject))
  /* [
    "foo",
    {...}
  ] ==> [
    "foo",
    {
      0:"baz",
      length:1,
      Symbol(Symbol.isConcatSpreadable): true
    }
  ] */
arrayLikeObject[Symbol.isConcatSpreadable] = true
console.log(initial.concat(arrayLikeObject))
  // ["foo", "baz"]

let otherObject = new Set().add('qux')
console.log(otherObject[Symbol.isConcatSpreadable])
  // undefined
console.log(initial.concat(otherObject))
  // ["foo", Set(1)] ==> [0: "foo", 1: Set(1) {"qux"}]
otherObject[Symbol.isConcatSpreadable] = true
console.log(initial.concat(otherObject))
  // ["foo"]

8. Symbol.iterator

  • 根据es规范该符号作为属性表示一个方法

    • 该方法返回对象默认迭代器，由for-of语句使用；即：该符号表示实现迭代器API的函数

    • for-of循环这样的语言结构利用该函数执行迭代器操作

      • 循环时调用以Symbol.iterator为键的函数
      • 默认该函数返回一个实现迭代器API的对象
      • 返回的对象是实现该API的 Generator
      class Foo {
  *[Symbol.iterator] () {}
}
let f = new Foo()
console.log(f[Symbol.iterator]())
// Generator {<suspended>} ==>
/***
__proto__: Generator
[[GeneratorLocation]]: VM269:2
[[GeneratorState]]: "suspended"
[[GeneratorFunction]]: ƒ* [Symbol.iterator]()
[[GeneratorReceiver]]: Foo
[[Scopes]]: Scopes[3]
*/

  • 技术上这个由 Symbol.iterator 函数生成的对象应通过其next()方法陆续返回值

    • 可通过显示调用next()方法返回
    • 可隐式通过生成器函数返回
      class Emitter {
  constructor (max) {
    this.max = max
    this.idx = 0
  }
  *[Symbol.iterator] () {
    while (this.idx < this.max) yield this.idx++
  }
}
function count () {
  let emitter =  new Emitter(5)
  for (const x of emitter) console.log(x)
}
count()
/**
  0
  1
  2
  3
  4
*/

9. Symbol.match

  • 根据es规范该符号作为属性表示正则表达式方法

    • 该方法用正则表达式匹配字符串
    • 由String.prototype.match()方法使用

  • 该方法使用Symbol.match为键的函数对正则表达式求值

    • 所有正则表达式实例默认该String方法的有效参数
      • 正则表达式原型上默认有该函数定义
        console.log(RegExp.prototype[Symbol.match])
// [Symbol.match]() {[native code]}
console.log('foobar'.match(/bar/))
// ["bar", index: 3, input: "foobar", groups: undefined]

  • 该方法传入非正则表达式值会将该值转为RegExp对象；如果需要改变此行为

    • 可通过方法直接使用参数，可重定义Symbol.match函数取代默认对正则表达式求值的行为
      • 从而使match()方法使用非正则表达式实例

  • Symbol.match函数接收一个参数

    • 调用match()方法的字符串实例

  • 返回值无限制

    class FooMatcher {
  static [Symbol.match] (target) {
    return target.includes('foo')
  }
}
console.log('foobar'.match(FooMatcher))
  // true
console.log('barbaz'.match(FooMatcher))
  // false

class StringMatcher {
  constructor (str) {
    this.str = str
  }
  [Symbol.match] (target) {
    return target.includes(this.str)
  }
}
console.log('foobar'.match(new StringMatcher('foo')))
  // true
console.log('barbaz'.match(new StringMatcher('qux')))
  // false

10. Symbol.replace

  • 根据es规范该符号作为属性表示正则表达式方法

    • 该方法替换字符串中匹配的子串
    • 由String.prototype.replace()方法使用

  • 该方法使用Symbol.replace为键的函数对正则表达式求值

  • 所有正则表达式实例默认该String方法的有效参数

    • 正则表达式原型上默认有该函数定义
      console.log(RegExp.prototype[Symbol.replace])
// f [Symbol.replace]() {[native code]}
console.log('foobar'.replace(/bar/, 'qux'))
// 'fooqux'
    • 该方法传入非正则表达式值该值会转为RegExp对象；如果需要改变此行为
      • 方法直接使用参数，可重定义Symbol.replace函数取代默认对正则表达式的求值行为
        • 从而使 replace()方法使用非正则表达式实例

  • Symbol.replace函数接收两个参数

    • 调用replace()方法的字符串实例
    • 替换字符串

  • 返回值无限制

    class FooReplacer {
  static [Symbol.replace] (target, replacement) {
    return target.split('foo').join(replacement)
  }
}
console.log('barfoobaz'.replace(FooReplacer, 'qux'))
  // "barquxbaz"

class StringReplacer {
  constructor (str) {
    this.str = str
  }
  [Symbol.replace] (target, replacement) {
    return target.split(this.str).join(replacement)
  }
}
console.log(
  'barfoobaz'.replace(
    new StringReplacer('foo'),
    'qux'
  )
)
  // "barquxbaz"

11. Symbol.search

  • 根据es规范，该符号作为属性表示正则表达式方法

    • 该方法返回字符串中匹配正则表达式的索引
    • 由Symbol.prototype.search()方法使用

  • 该方法使用以Symbol.search为键的函数对正则表达式求值

  • 所有正则表达式实例默认是该String的有效参数

    • 正则表达式原型上默认有该函数定义
      console.log(RegExp.prototype[Symbol.search])
// f [Symbol.search]() {[native code]}
console.log('foobar'.search(/bar/))
// 3

  • 该方法传入非正则表达式值该值转为RegExp对象；若需要改变此行为

    • 方法直接使用参数，可重定义Symbol.search函数取代默认对正则表达式求值行为
    • 从而让search()方法使用非正则表达式实例

  • Symbol.search函数接收一个参数

    • 调用match()方法的字符串实例

  • 返回值无限制

    class FooSearcher {
  static [Symbol.search] (target) {
    return target.indexOf('foo')
  }
}
console.log('foobar'.search(FooSearcher))
  // 0
console.log('barfoo'.search(FooSearcher))
  // 3
console.log('barbaz'.search(FooSearcher))
  // -1

class StringSearcher {
  constructor (str) {
    this.str = str
  }
  [Symbol.search] (target) {
    return target.indexOf(this.str)
  }
}
console.log('foobar'.search(new StringSearcher('foo')))
  // 0
console.log('barfoo'.search(new StringSearcher('foo')))
  // 3
console.log('barbaz'.search(new StringSearcher('qux')))
  // -1

12. Symbol.species

  • 根据es规范，该符号作为属性表示一个函数值

    • 该函数作为创建派生对象的构造函数

  • 该属性常用于内置类型

    • 用于对内置类型实例方法的返回值暴露实例化派生对象的方法

  • 用于Symbol.species定义静态获取器（getter）方法

  • 可覆盖新创建实例的原型定义

    class Bar extends Array {}
class Baz extends Array {
  static get [Symbol.species] () {
    return Array
  }
}
let bar = new Bar()
console.log(bar instanceof Array)
  // true
console.log(bar instanceof Bar)
  // true
bar = bar.concat('bar')
console.log(bar instanceof Array)
  // true
console.log(bar instanceof Bar)
  // true

let baz = new Baz()
console.log(baz instanceof Array)
  // true
console.log(baz instanceof Baz)
  // true
baz = baz.concat('baz')
console.log(baz instanceof Array)
  // true
console.log(baz instanceof Baz)
  // false

13. Symbol.split

  • 根据es规范，该符号作为属性表示正则表达式方法

    • 该方法在匹配正则表达式的索引位置拆分字符串
    • 由String.prototype.split()方法使用

  • 该方法使用以Symbol.split为键的函数对正则表达式求值

  • 所有正则表达式实例默认是这个String方法的有效参数

    • 正则表达式原型上默认由该函数定义
      console.log(RegExp.prototype[Symbol.split])
// f [Symbol.split] () {[native code]}
console.log('foobarbaz'.split(/bar/))
// ["foo", "baz"]

  • 给该方法传入非正则表达式值，值转为RegExp对象；若需要改变此行为

    • 方法直接使用参数，可重定义Symbol.split函数取代默认对正则表达式求值行为
    • 从而让split()方法使用非正则表达式实例

  • Symbol.split 函数接收一个参数

    • 调用match()方法的字符串实例

  • 返回值无限制

    class FooSpliter {
  static [Symbol.split] (target) {
    return target.split('foo')
  }
}
console.log('barfoobaz'.split(FooSpliter))
  // ["bar", "baz"]

class StringSplitter {
  constructor (str) {
    this.str = str
  }
  [Symbol.split] (target) {
    return target.split(this.str)
  }
}
console.log('barfoobaz'.split(new StringSplitter('foo')))
  // ["bar", "baz"]

14. Symbol.toPrimitive

  • 根据es规范，该符号作为属性表示一个方法

    • 该方法将对象转为相应原始值由ToPrimitive抽象操作使用

  • 很多内置操作都会尝试强制将对象转为原始值

    • 包括字符串、数值、未指定的原始类型

  • 对一个自定义对象实例，通过该实例的Symbol.toPrimitive属性上定义一个可改变默认行为的函数

    • 根据提供给该函数的参数（string、number、default）可控制返回的原始值

      class Foo {}
let foo = new Foo()
console.log(3 + foo)
  // "3[object Object]"
console.log(3 - foo)
  // "NaN"
console.log(String(foo))
  // "[object Object]"

class Bar {
  constructor () {
    this[Symbol.toPrimitive] = function (hint) {
      switch (hint) {
        case 'number':
          return 3
        case 'string':
          return 'string bar'
        case 'default':
        default:
          return 'default bar'
      }
    }
  }
}
let bar = new Bar()
console.log(3 + bar)
  // "3 default bar"
console.log(3 - bar)
  // 0
console.log(String(bar))
  // "string bar"

15. Symbol.toStringTag

  • 根据es规范，该符号作为一个属性表示一个字符串

    • 该字符串用于创建对象的默认字符串描述
    • 由内置方法 Object.prototype.toString使用

  • 通过toString方法获取对象标识时

    • 检索由Symbol.toString指定的实例标识符,默认为 “Object”

    • 内置类型指定该值，但自定义类实例需明确定义

      let s = new Set()
console.log(s)
  // Set(0) {}
/**
[[Entries]]
No properties
size: 0
__proto__: Set
*/
console.log(s.toString())
  // [object Object]
console.log(s[Symbol.toStringTag])
  // Set

class Foo {}
let foo = new Foo()
console.log(foo)
  // Foo {}
console.log(foo.toString())
  // [object Object]
console.log(foo[Symbol.toStringTag])
  // undefined

class Bar {
  constructor () {
    this[Symbol.toStringTag] = 'Bar'
  }
}
let bar = new Bar()
console.log(bar)
  // Bar {} ==> Bar {Symbol(Symbol.toStringTag): "Bar"}
console.log(bar.toString())
  // [object Bar]
console.log(bar[Symbol.toStringTag])
  // Bar

16. Symbol.unscopables

  • 根据es规范，该符号作为属性表示对象
    • 该对象所有的以及继承的属性，都会从关联对象的with环境中绑定排除
  • 设置该符号并使其映射对应属性的键值为true即可阻止该属性出现在with环境绑定中
    let o = { foo: 'bar' }
with (o) { console.log(foo) }
  // bar
o[Symbol.unscopables] = { foo: true }
with (o) { console.log(foo) }
  // ReferenceError: foo is not defined

**注意：不推荐使用with，因此也不推荐使用该方法 **

总结：

symbol类型作用

  • 确保对象属性使用唯一标识符，避免属性冲突

描述

  • 虽然和私有属性类似，但符号并不是为提供私有属性增加；尤其是因Object Api提供方法可更方便发现符号属性
  • 符号用于创建唯一记号，用于非字符串形式的对象属性
  • 没有字面量语法
  • 不能和 new 关键字一起使用成为构造函数

方法

  • Symbol.for() ==> 重用已有符号
  • Symbol.keyfor() ==> 查询全局注册表
  • Symbol.asyncIterator ==> 返回异步迭代器对象
  • Symbol.iterator ==> 实现迭代器API的函数
  • Symbol.hasInstance ==> 确定对象实例的原型链上是否有原型
  • Symbol.isConcatSpreadable ==> 类数组对象拼接为数组实例
  • Symbol.match ==> 正则表达式匹配字符串
  • Symbol.replace ==> 正则表达式 替换字符串中匹配的子串
  • Symbol.search ==> 返回字符串中匹配正则表达式的索引
  • Symbol.species ==> 作为创建派生对象的构造函数
  • Symbol.split ==> 在匹配正则表达式的索引位置拆分字符串
  • Symbol.toPrimitive ==> 将对象转为相应的原始值
  • Symbol.toStringTag ==> 创建对象的默认字符串描述
  • Symbol.unscopables ==> 该对象所有的及继承的属性都从关联对象的with环境中绑定排除（不推荐）
