命令

阅读量: 静思 2019-10-11 10:23:00
Categories： 工具类 Tags： git

1. 常用命令

  • pwd (print working directory) 查看当前所在目录
  • cd (change directory) 切换目录
  • ls (list) 查看当前目录下内容
  • clear 清屏
  • mkdir (make directory) 创建目录
  • touch 创建文件
  • cat 查看文件内容 一次性输出
  • less 查看文件内容 部分输出
    • 回车 向下一行
    • 空格 向下一页
    • b 向上一页
    • q 退出
  • rm (remove) 删除文件；如：rm index.html、rm -rf blog
  • rmdir (remove directory) 删除空文件夹
  • mv (move) 移动文件或重命名
  • cp (copy) 拷贝文件
  • echo ‘内容’ > 文件 将内容覆盖输出到文件
  • echo ‘内容’ >> 文件 将内容追加输出到文件

2. git三大区域

  • 工作目录
    • 存放项目代码的目录
  • 暂存区域
    • 临时存储更改过的文件
    • 作用：防止工作目录中代码丢失
  • 代码仓库
    • 项目开发完成一个版本时，将代码形成版本提交到仓库
    • 将本地工作目录代码存储到代码仓库

3. git常用命令

  • 配置 git 用户名和邮箱

    git config –global user.name zhangsan
    git config –global user.email zhangsan@163.com

  • 查看当前git配置

    git config –list

  • 初始化git仓库

    git init

  • 查看当前仓库状态

    git status

  • 将工作目录的更改添加到暂存区域

    git add

    • git add * ==> 添加所有
    • git add 文件名 ==> 添加指定文件
  • 将暂存区域代码提交到代码仓库形成版本

    git commit -m ‘版本描述信息’

  • 查看工作目录提交历史

    git log

  • 回退版本（将暂存区域文件恢复到工作目录）

    git rm –cached 文件列表

    • 说明：
      • 必须保证工作目录中代码与暂存区域代码一致
      • 工作目录有该文件且暂存区域没有该文件并不被git管理
  • 暂存区域中的文件覆盖工作目录中的文件

    git checkout – 文件列表

    • 说明:
      • 暂存区域和工作目录都有该文件且被git管理
  • 回滚到本地仓库中的特定版本并覆盖暂存区域和工作目录

    git reset –hard commitID

  • 查看分支

    git branch

  • 创建分支

    git branch 分支名称

  • 切换分支

    git checkout 分支名称

  • 创建并切换分支

    git checkout -b 分支名称

  • 删除分支（如果分支未合并 不允许删除）

    git branch -d 分支名称

  • 强制删除分支

    git branch -D 分支名称

  • 合并分支

    git merge 来源分支

  • 初始化一个裸露仓库（公共代码仓库）

    git init -bare

  • 推送代码到远程仓库

    git push 远程仓库地址 本地分支名称:远程分支名称

  • 拉取远程仓库代码带本地（拉取最新）

    git pull 远程仓库地址 远程分支名称:本地分支名称

  • 远程仓库地址创建别名

    git remote add 别名 远程仓库地址

  • 查看远程地址详细信息

    git remote -v

  • 查看当前别名所对应的远程仓库地址

    git remote show 别名

  • 删除当前别名所对应的远程仓库地址

    git remote remove 别名

  • 获取远程仓库代码（拉取所有版本到本地）

    git clone 远程仓库地址 项目名称

    • 使用场景：
      • 加入已有项目，需要先拉取所有版本到本地再进行开发

4. 冲突修复

  • 模拟冲突：
    1. a 和 b 分别克隆代码到本地
    2. a 修改 demo 文件并提交远程仓库
    3. b 修改 demo 文件提交
    4. 因 a 先提交一个版本到远程仓库，导致远程仓库代码比 b 的本地代码更新一个版本
    5. b 需要先拉取远程仓库代码，保证远程仓库和本地代码一致，再进行代码推送
  • 冲突产生原因：
    • a 和 b 同时修改一个文件
  • 解决方案：
    • 后提交者 先拉取后推送
  • 多人协作开发免登录操作
    • ssh-keygen

5. git忽略清单

  • .gitignore 文件

6. git使用流程

  1. 配置账户信息

    git config –global user.name 用户名称
    git config –global user.email 用户邮箱
    git config –list 查看当前git配置

  2. 初始化本地仓库并查看状态

    git init
    git status

  3. 将工作目录文件提交到暂存区域

    git add 文件列表
    git add .添加到工作目录中所有文件到暂存区

  4. 将暂存区域代码提交到本地仓库形成版本

    git commit -m 版本描述

  5. 本地仓库代码提交到远程仓库更新版本

    git push 远程仓库地址 本地分支名称:远程分支名称
