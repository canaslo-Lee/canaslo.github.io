hexo

阅读量: 静思 2021-05-02 02:56:33
Categories： 工具类 Tags： hexo

一定要使用npm
打开命令行快捷键 win + r 输入cmd 按回车
命令行关闭方式：输入 exit 按回车

准备工作

安装npm及node

  • 链接： https://nodejs.org/zh-cn/
  • 说明：选择长期支持版点击下载然后安装
  • 安装完成后打开命令行输入命令检查版本；如果出现版本号则说明安装完成

    npm –version
    node –version

安装git

  • 链接：https://git-scm.com/
  • 说明：找到右边的电脑里边有 Download开头的按钮点击下载
  • 安装完成后右键出现的工具里会有git bash的选项

安装编辑器

下列编辑器选择其中一个即可

  • sublime下载并安装
    • 链接：https://www.sublimetext.com/
  • vscode下载并安装
    • 链接：https://code.visualstudio.com/

安装hexo

桌面打开npm并输入

npm install -g hexo
安装完成后输入命令检查是否安装完成
hexo –version

生成ssh公钥

  • 桌面右键选择 git bash here
  • 输入以下命令

    ssh-keygen -t rsa -C “你的邮箱账号@email.com”

    • 根据你的需要决定是否根据提示配置密码
  • 创建博客公钥
    • 输入以下命令

      ssh-keygen -t rsa -C “你的邮箱账号@email.com” -f ~/.ssh/你自行决定的公钥名称

      • 根据你的需要决定是否根据提示配置密码

仓库配置

  • 登录github；如果没有自行创建
  • 新建一个代码仓库
    • 仓库名必须是 你的github用户名.github.io
    • 仓库为公开仓库
    • 其他根据需求自行设置
  • 创建好后查看左上角你的github用户名/你的github用户名.github.io 中github用户名是否一致
    • 一致进行下一步
    • 不一致 点击settings 在 repository name 中修改为一致 并点击 Rename
  • 点击右上角你的头像选择settings
  • 左边列表点击 SSH and GPG keys
  • 点击在 ssh keys 右侧的 new ssh key 按钮
  • 在title 输入名称（这里使用的是 id_rsa公钥最好；为方便标识 我使用的名称是 id_rsa，便于和公钥对应）
  • win + e 打开文件管理器；进入c\用户`你的管理员名称文件夹`
  • 找到.ssh文件 如果没有点击查看 勾选隐藏的项目
  • 找到id_rsa.pub 右键打开方式找到你的编辑器打开（直接点是打不开的）
  • ctrl + a 全选；然后 ctrl + c 复制；在 github配置公钥的界面的 key里粘贴
  • 点击你的用户名、点击你的仓库名进入仓库、点击settings
  • 找到 Deploy keys 点击 add deploy key；自行决定输入该公钥的名称
  • 打开.ssh文件 将创建的博客公钥.pub用编辑器打开（如果真有小白，看一下上一个公钥的打开方式，直接点是打不开的）
  • 全选、复制、粘贴进key里
  • 勾选、勾选、勾选Allow write access；点击 add key按钮

创建项目

在项目文件夹父级输入命令；示例：d\project\blog

  • 进入到 d\project 打开npm 输入命令进行初始化配置

    hexo init blog

  • 配置完成后通过编辑器将blog文件夹打开
  • 找到config.yml文件点击打开并拉到底部找到 deploy 将下列代码覆盖进去；其他配置可自行设置
    deploy:
  type: git
  repository: git@github.com:`你的github用户名`/`你的github用户名`.github.io.git
  branch: master
  • repository 这个地址的位置在github 仓库 点击 code 进入后 右侧有个绿色 的code按钮
  • 点击按钮 选择 SSH，下边有git@github.com开头的路径 点击右侧记事本按钮复制在repository后边粘贴即可

命令

  • 命令
    • 清除缓存

      hexo clean

    • 构建项目

      hexo g

    • 本地预览
      hexo s
    • 上传到远程仓库

      hexo d

  • 打开cmd 切换到当前目录
    • win + r
    • 输入 cmd 回车
    • 示例：在d盘 就输入d: 回车
    • 然后cd 你项目的路径; 示例：d\project\blog
      • cd project 回车
      • cd blog 回车
  • 输入命令

    npm install hexo-deployer-git
    npm install hexo-generator-cname

  • 安装完成后输入

    hexo clean
    hexo g
    hexo d

  • 如果最后显示的是 deploy done: git 则说明提交到仓库成功
  • 可在仓库查看 code 中 中间会显示提交的时间日期
  • 在浏览器地址栏输入你的github用户名.github.io就能看到博客了

主题

  • 链接：https://hexo.io/themes/
  • 说明：
    • 选择你喜欢的主题点击主题名称进入github 复制https链接地址
    • 打开你的项目文件夹找到themes文件夹进入,右键选择git bash here
    • 在git bash 中输入 git clone 鼠标移入右键选择粘贴 然后回车
    • 编辑器中打开_config.yml文件 找到 themes 修改为自己下载下来的主题名称
      # Extensions 扩展名
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: 这里输入你选择主题名称
    • 然后输入命令提交即可

      hexo clean
      hexo g
      hexo d

安装配置时踩得坑

  • 坑1：

    • hexo d 出现错误：

      git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
add th repository exists.
FATAL {
  err: ERROR: Spawn failed
  ....
}

    • 解决方案：

      • 谷歌各种办法无解
      • 最后小机灵一抖给用户加了一个ssh

  • 坑2：

    • 仓库名和用户名不一致
    • 解决方案：
      • 修改仓库名

  • 坑3：

    • 自定义域名绑定后在Source文件夹内添加CNAME文件输入域名创建无效
    • 解决方案
      • 嗯，又是罗里吧嗦的各种谷歌浪费时间
      • 通过各种放文件位置，查官方文档找到问题原因了
      • 事实证明，官方文档有时候比看谷歌啥的他们比比好使多了
      • 解决方案是
        • 项目根目录下_config.yml文件中 url要改为自定义域名
          # URL
## Set your site url here. For example, if you use GitHub Page, set url as 'https://username.github.io/project'
# 之前的地址
# url: https://*.github.io/
# 修改后的地址
url: https://lakvovs.com.cn
