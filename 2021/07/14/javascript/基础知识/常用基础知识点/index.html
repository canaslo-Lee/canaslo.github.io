常用基础知识点

阅读量: 静思 2021-07-14 10:18:12
Categories： javascript Tags： js基础

js是什么

  • 由BOM、DOM、ECMAScript组成的一种解释性脚本语言、用于向结构层添加交互、可直接在结构层书写，写成单独的js文件有利于结构行为分离

区分是否严格模式

  • 是否存在 “use strict”

严格模式优点

  • 保证代码运行安全
  • 提高编译效率，增加运行速度

严格模式的区别：

  • 严格模式中运行的结果可能与松散模式下运行的结果不同
  • 有些松散模式下运行的语句在严格模式中不能运行

严格模式的限制

  • 禁止删除变量、对象、函数、不允许删除的属性
  • 禁止使用未声明变量、八进制、转义字符、getter方法读取的属性进行赋值
  • 禁止变量重名、对只读属性赋值、this关键字指向全局对象（使用构造函数时如果忘记加new，this将报错）
  • 变量名不能使用eval、arguments字符串
  • 由于安全原因，作用域eval()创建的变量不能被调用

变量定义、声明

  • var定义变量（es5）、let定义变量（es6）、const定义常量（es6）；以分号结束声明

数据类型

  • string、boolean、object、array、number、undefined、null、symbol（es6）

校验数据的类型

  • typeof ‘string’ === true ? true : false
  • ‘变量’ instanceOf ‘数据类型’
  • Object.prototype.toString.call(object) === ‘[object Array]’
  • Array.isArray(object)
  • ‘string’.constructor === String
  • object.constructor === Array

数据类型转换

  • 强制：parseInt、parseFloat、Number、String、Boolean
  • 隐式：==、!!、+、-

== 和 === 的区别

  • 数据类型是否一致
  • 是否自动转换类型
  • == 自动转换数据类型，不判断数据类型
  • === 不自动转换数据类型，判断数据类型

语句

  • 判断语句
    • if、if·else、while、do·while、switch
  • 循环语句
    • for、forEach、for·of

数组方法

  • unshift()：头部添加
  • push()：尾部添加
  • pop()：删除并返回数组中最后一个
  • shift()：删除并返回数组中第一个
  • sort()：对数组排序
  • concat()：合并数组并返回
  • reverse()：反转数组
  • slice(start,end)：从数组中返回选中元素
    • start：必填，开始选中位置，值为负数则从数组尾部向前截取
    • end：选填，结束选中位置，值为负数则从数组尾部向前截取，未指定则截取所有
  • splice(index, length, options)：插入、删除
    • index：必填，整数，添加|删除的索引
    • length：必填，整数，为0不删除，不为0则删除从index开始向后length个数据
    • options：选填，添加的项
  • join(separator)：将数组所有元素放入一个字符串内
    • separator：选填，指定使用的分隔符，默认逗号

字符串方法

  • anchor(name)：创建html锚
  • charAt(index)：返回指定位置字符；必填，字符在字符串中的索引
  • charCodeAt(index)：返回指定字符的Unicode编码
  • concat()：拼接字符串
  • indexOf()：检索字符串
  • lastIndexOf()：从后向前检索字符串
  • slice(start, end)：截取并返回新字符串，值为负数则从结尾开始截取
    • start：开始索引
    • end：结束索引，未指定则截取到原字符串结尾
  • substr(start， length)：截取从指定索引开始指定个数的字符，值为负数则从结尾开始截取
    • start：必填，开始截取的索引，数值
    • length：选填，截取个数，忽略则返回从start开始到字符串结尾
  • substring(start, stop)：截取两个非负整数之间的字符
    • start：必填，截取的第一个字符在字符串中的索引
    • stop：选填，截取的最后一个字符在字符串中的索引，忽略则返回从开始到字符串结尾的子串
  • split(separator，MaxLength)：字符串分割为数组
    • separator：必填，字符串或正则
    • MaxLength：选填，指定返回数组的最大长度
  • replace(regexp|substr，replacement)：替换指定子串
    • regexp|substr：必填，要替换的正则|子串
    • replacement：必填，替换的子串
  • search(regexp)：查找匹配的子串起始索引，找不到返回 -1；忽略大小写：参数结尾追加 i
  • valueOf()：返回字符串对象的原始值
  • 大小写转换
    • toLowerCase()：转为小写
    • toLocalLowerCase()：转为小写
    • toUpperCase()：转为大写
    • toLocalUpperCase()：转为大写
  • 字符串显示方式
    • big()：大号字体
    • smail()：小号
    • strike()：删除线
    • blink()：闪动
    • italics()：斜体
    • bold()：粗体
    • fixed()：打字机文本
    • link()：链接
    • fontcolor(color)：指定颜色；必填，值为颜色名、rgb、十六进制
    • fontsize(size)：指定大小；必填，值为1~7的数字
    • fromCharCode(numX, numX)：通过unicode编码创建字符串；必填，一个及以上个unicode编码

数字方法

  • Number()：值转为数字，无法转换则返回NaN；参数为Date则返回1970年1月1日至今的毫秒数
  • parseInt(string, radix)：值转为数字并取整；参数二忽略或为0则使用十进制，以0x、0X开头使用十六进制，小于2或大于36返回NaN
  • Math.ceil()：向上取整（有小数则进1）
  • Math.floor()：向下取整（忽略小数）

js面向对象的几种方式

  1. 工厂函数
    const lev = () => '工厂函数'
function Parent () {
 const Child = new Object()
 Child.name = 'name'
 Child.age = '4'
 Child.lev = lev
 return Child
}
console.log(Parent().name, Parent().lev()) // name, 工厂函数
  2. 构造函数
    const lev = () => 'return string'
function Parent () {
 this.name = 'name'
 this.age = '30'
 this.lev = lev
}
console.log(new Parent().name, new Parent().lev()) // name, return string
  3. 原型模式
    const lev = () => 'return string'
function Parent () {}
Parent.prototype.name = 'name'
Parent.prototype.age = '30'
Parent.prototype.lev = lev
console.log(new Parent().name, new Parent().lev()) // name, return string
  4. 混合构造函数，原型方式
    function Parent () { this.name = 'name'; this.age = 4 }
/* 此处赋值禁止使用箭头函数，会导致this指向改变问题 */
Parent.prototype.lev = function () {return this.name }
console.log(new Parent().lev()) // name
  5. 动态原型
    function Parent () {
 this.name = 'name'
 this.age = 4
 if (typeof Parent._lev === 'undefined') {
   Parent.prototype.lev = () => this.name
   Parent._lev = true
 }
}
console.log(new Parent().lev()) // name

this指向

  • 函数调用指向window
  • 对象调用指向对象实例
  • 构造函数调用指向其创建出的对象
  • apply调用指向该方法的第一个参数

this作用域

  • js作用域有两种：全局：其实是window对象的属性；局部：只在函数中通过变量声明才生效，为避免作用域问题，变量声明时需使用变量关键字，变量在全局作用域中声明的变量为全局变量，局部作用域声明的为局部变量

原型链概念

  • 每个实例对象都存在一个__proto__属性，该属性指向其原型对象，此实例对象的构造函数存在原型属性prototype，与实例的__proto__属性指向一致

原型链构成

  • 对象查找某个属性时，自身没有会根据__proto__向其原型查找，再没有则继续向下查找直到指向null；该过程形成的链式结构称为原型链

继承是什么

  • 继承更像是函数的功能用法，如何用函数做到复用、组合，这些和使用继承的思路是一致的
  • 原型式继承用其函数复制内部对象的一个副本，这样不仅可继承内部对象的属性还可以把函数、对象（内部对象的返回）随意调用、添加属性；修改参数即可改变原型对象，新增属性不会相互影响

继承的实现方式

  1. 原型链继承 - 重点：让新实例原型等于父类实例 - 特点：实例可继承构造函数、父类构造函数、父类原型属性；新实例不继承父类实例属性 - 缺点： 新实例无法向父类构造函数传参、继承方式单一；所有新实例都共享父类实例属性，一个实例修改原型属性，另一实例同步修改 - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function Per () { this.name = 'ker' }
Per.prototype = new Person() // 主要
let per1 = new Per()
console.log(per1.age) // 10
// 判断元素是否在另一个元素的原型链上
console.log(per1 instanceof Person) // true

  2. 借用构造函数继承 - 重点：用call()和apply()将父类构造函数引入子类函数 - 特点：只继承父类构造函数的属性，不能继承父类原型属性；解决原型链继承缺点，可继承多个构造函数，可在子实例向父类实例传参 - 缺点：只能继承父类构造函数的属性，无法实现构造函数复用，每次使用都需重新调用；每个新实例都有父类构造函数的副本，臃肿 - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function Con () {
  Person.call(this, 'jer')
  this.age = 12
}
console.log(new Con().name) // jer
console.log(new Con().age) // 12
console.log(new Con() instanceof Person) // false

  3. 组合继承（组合原型链和构造函数继承） - 重点：结合原型链和构造函数继承的优点，传参、复用 - 特点：可继承父类原型上的属性，可传参、复用；每个新实例引入的构造函数属性是私有的 - 缺点：调用两次父类构造函数，子类的构造函数会替代原型上的父类构造函数 - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function subType (n) { Person.call(this, n) /*借用构造函数模式*/ }
subType.prototype = new Person() // 原型链继承
// 继承构造函数属性
console.log(new subType('gar').name) // 'gar'
// 继承父类原型的属性
console.log(new subType('gar').age) // 10

  4. 原型式继承 - 重点：用函数包装一个对象并返回该函数调用，该函数就成了可随意增加属性的实例或对象，Object.create()就是这个原理 - 特点：类似复制一个对象，用函数包装 - 缺点：所有实例都继承原型上的属性，无法实现复用（新实例属性都是后添加） - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function content (obj) {
  function F () {}
  F.prototype = obj // 继承传入参数
  return new F() // 返回函数对象
}
let sup = new Person() // 获取父类实例
console.log(content(sup).age) // 10 继承父类函数属性

  5. 寄生式继承 - 重点：在原型式继承外面套了个壳子 - 特点：无创建自定义类型，因只是套了一个壳子返回该对象，该函数成为创新的新对象 - 缺点：未使用原型，无法复用 - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function content (obj) {
  function F () {}
  F.prototype = obj // 继承传入参数
  return new F() // 返回函数对象
}
let sup = new Person() // 获取父类实例
// ↑ 是原型式继承，给原型式继承套个壳子再传递参数
function subObject (obj) {
  let sub = content(obj)
  sub.name = 'gar'
  return sub
  // 此处不能直接返回；直接返回导致 typeof sup1 的数据类型是 string； sup1.name 的返回是undefined
}
let sup1 = subObject(sup)
// ↑ 该函数声明后成为可增加属性的对象
console.log(typeof subObject) // function
console.log(typeof sup1) // 'object'
console.log(sup1.name) // 'gar' 返回sub对象继承sub属性

  6. 寄生组合式继承 - 寄生：在函数内返回对象然后调用 - 组合：函数的原型等于另一个实例；函数中用apply或call引入另一个构造函数，可传参 - 重点：修复组合继承的问题 - demo

    function Person (name) {
  this.name = name
  this.sum = function () { console.log(this.name) }
}
Person.prototype.age = 10 // 向构造函数添加原型属性
function content (obj) {
  function F () {}
  F.prototype = obj // 继承传入参数
  return new F() // 返回函数对象
}
let con = content(Person.prototype)
// con实例（F实例）的原型继承父类函数的原型
// 上述更像原型链继承，但只继承了原型属性

// 组合式继承
// 解决组合式两次调用构造函数属性缺点
function sub () { Person.call(this)/* 继承父类构造函数属性*/ }

// 重点
sub.prototype = con // 继承con实例
con.prototype = sub // 一定要修复实例
let sub1 = new sub()
// sub 实例继承构造函数属性、父类实例、con的函数属性
console.log(sub1.age) // 10

闭包是什么

  • 闭包是函数内部和函数外部链接的桥梁，可读取其他函数内部变量的函数；当函数返回值为另一个函数时，返回的函数若调用其父级函数内部的变量且返回该函数在外部被执行时，会产生一个闭包环境，具体指的是外部高阶函数
  • 闭包是函数和声明该函数的此法环境组合，内部函数对外部函数作用域内变量的引用；函数内属性在函数执行期间有生命周期；闭包内的函数私有化变量，完成数据封装，类似面向对象，子函数被父级函数之外的的变量引用会形成闭包

闭包作用

  • 可组曲其他函数内部变量，让这些变量的值始终在内存中，不被外层函数调用后自动清除
  • 保存自己私有变量，通过提供的接口、方法给外部使用，外部不能直接访问该变量

闭包原理

  • 闭包只在被调用时执行，所以可被用于定义控制结构
  • 多个函数可使用同一环境，这使得他们可以通过改变那个环境相互交流

闭包使用场景

  • 采用函数引用方式的setTimeout调用，将函数关联到对象的实例方法，封装相关功能集

闭包优点

  • 变量长期在内存中存储，避免全局变量污染，私有成员的存在

闭包缺点

  • 数据长期存储在内存中会增加内存使用量，若使用不当将导致内存泄漏，解决办法是函数退出前将不使用的局部变量全部删除
  • 闭包会在父级函数外部改变父级函数内部变量的值
  • 当把父级函数当对象使用时，闭包当作其公用方法
  • 内部变量当作其私有属性时不要随意改变父级函数内部变量的值

闭包特性

  • 作为函数变量的一个引用，函数返回时，其处于激活状态；闭包就是当一个函数返回时，并没有释放资源的栈区

闭包对页面的影响

  • 通过使用闭包，可以做很多事情，如：模拟面向对象的代码风格；
  • 更优雅、简洁的表达出代码
  • 在某些方法提升代码执行效率

事件委托是什么

  • 自己需要监控的事件交给父级监控

事件委托的优点

  • 提升性能，多列表注册事件，使用事件委托，只需要将事件给父级注册一次即可
  • 新增元素也由之前的事件操控

监听事件的方法

  • 行内绑定：onclick
  • element.onclick：通过DOM元素为DOM绑定事件
  • addEventListener(eventName, eventFunction, boolean[true：事件捕获触发；false：事件冒泡触发])：ie8+使用
  • attachEvent(eventName, eventFunction)：ie8-使用
    • 该函数在事件冒泡阶段触发；可绑定多个事件函数，执行顺序与绑定相反
  • removeEventListener()：事件解绑
    • 参数与添加事件函数一致；第二参数（事件函数）必须指向同一个地址
    • 事件函数应保存在一个变量中；传入匿名函数无法移除该事件监听器

什么是事件冒泡

  • 在一个对象上触发某类事件（如：单击事件）
    • 如果该对象定义了此事件的处理程序，则该事件的处理程序被调用
    • 如果未定义该事件处理程序或返回true，则该事件会向父级对象传播
    • 从事件对象自身向上传播直到其被处理，或者向上到达对象顶层对象，即document/window对象
    • 向上传播的过程中父级对象所有同类事件均被触发

阻止事件冒泡的方法

  • return false：事件处理过程中阻止事件冒泡、阻止默认行为
  • event.stopPropagation()：事件处理过程中阻止事件冒泡，未阻止默认行为，如：超链接跳转可触发
  • event.preventDefault()：事件处理过程中不阻止事件冒泡，但阻止默认行为，如：执行所有弹框，不执行超链接跳转

事件绑定的普通事件有什么区别

  • 普通事件不支持添加多个事件，最下面的事件会覆盖上面的，无法取消
  • 事件绑定（addEventListener）可添加多个事件，ie低版本需要使用attachEvent，支持事件冒泡和事件捕获

IE和DOM事件流的区别

什么是事件流

  • 从页面中接收事件的顺序

事件流种类

  • 事件冒泡流（IE事件流）：事件开始时由事件自身节点接收，逐级向上传播到文档节点，从内向外
  • 事件捕获流（Netscape事件流）：文档节点最先接收到事件，事件自身最后接收到事件，事件到达预定目标前捕获到，从外向内
  • DOM事件流：
    • 包含三个阶段：事件捕获阶段、处于目标阶段、事件冒泡阶段
    • 首先发生事件捕获，为截获事件提供机会，然后实际目标接收事件，最后冒泡阶段，该阶段对事件做出相应处理，从外向内，然后从内向外

IE事件流和DOM事件流的区别

  • 执行顺序
    • IE：从内向外
    • Netscape：从外向内
    • DOM：从外象内，然后从内向外
  • 事件侦听函数
    • IE：
      • 绑定：attachEvent(事件名，回调处理函数)
      • 解绑：datachEvent(事件名，回调处理函数)
    • DOM
      • 绑定：addEventListener(事件名，回调处理函数，Boolean表示回调函数在事件传递中的捕获阶段被调用还是在冒泡阶段被调用)
      • 解绑：removeEventListener(事件名，回调处理函数，Boolean表示回调函数在事件传递中的捕获阶段被调用还是在冒泡阶段被调用)
  • this指向不同
    • IE：注册的处理函数调用时this指向window；事件名以on开头
    • DOM：事件名不以on开头
      • 注册的处理函数调用时this指向被绑定的DOM元素
      • 处理函数第一个参数是event对象
      • 可在DOM元素上绑定多个事件处理函数

js本地对象、内置对象、宿主对象

定义：

  • 本地对象（内部对象/native object）：
    • 本地对象就是ECMA-262定义的类，是引用类型
    • 运行中动态创建的对象需要使用 new 关键字
  • 内置对象（Build-in object）：
    • 由es实现提供，独立于宿主环境的所有对象
    • 在es开始执行时出现，即引擎渲染初始化阶段被创建的对象
    • 不必明确实例化内置对象，已被实例化
  • 宿主对象（host object）：
    • 由es实现的宿主环境提供的对象
    • 包含宿主提供、自定义类对象两大类
    • es官网未定义的对象都属于宿主对象
    • 所有非本地对象都是宿主对象
    • 原理：
      • 宿主框架通过某种机制注册到es引擎中的对象，如宿主浏览器会向es注入window对象，构建其实现js

        内容（包含）：

  • 本地对象：

  • 内置对象：
    • Global（全局对象）、Math
  • 宿主对象：
    • 由宿主提供的对象
    • 浏览器window对象及其所有子对象，如：BOM、DOM等

    • 什么是宿主对象：
      • web中，es的宿主是网页运行环境，即操作系统和浏览器
      • 所有非本地对象都是宿主对象，即由es实现的宿主环境提供的对象
  • 全局对象
    • 一个是es提供的Global对象
    • 一个是宿主提供，如：浏览器中的window、node中的Global。所以浏览器中全局对象是Global+window
    • 通常es提供的Global对象是不存在的，无具体对象

什么是内存泄漏（memory leak）

  • 不再使用的内存未及时释放就是内存泄漏

内存泄漏的缺点

  • 程序运行需使用内存，不及时释放导致内存占用越来越高，轻则影响性能，重则导致进程崩溃

垃圾回收机制

  • 大多数语言提供的自动内存管理，以减轻开发者负担，被称为垃圾回收机制（garbage collector）

垃圾回收方法

  • 引用计数
    • 语言引擎有一张引用表，保存内存内所有的资源的引用次数
    • 当一个值的引用次数为0，就表示该值不再使用，可释放该块内存
  • 标记清除

导致内存泄漏的原因

  • 一个值不再需要，但是引用表内的引用数不为0，导致垃圾回收机制无法释放该块内存，从而导致内存泄漏

内存泄漏的识别方法

  • 连续五次垃圾回收后，内存占用递增时就存在内存泄漏，此时需实时查看内存占用
  • 如果内存占用基本平稳，接近水平，则不存在内存泄漏，反之就存在内存泄漏

weakMap

  • es6新增weakSet、weakMap两种数据结构
  • 其对值的引用都不计入垃圾回收机制
  • 名字中的weak表示这是弱引用

描述cookies、sesstionStorage、localStorage的区别

  • 共同点：都是保存在浏览器端，且同源的。
  • 区别：
    1. cookie数据始终在同源的http请求中携带（即使不需要），即cookie在浏览器和服务器间来回传递。而sessionStorage和localStorage不会自动把数据发给服务器，仅在本地保存。cookie数据还有路径（path）的概念，可以限制cookie只属于某个路径下。
    2. 存储大小限制也不同，cookie数据不能超过4k，同时因为每次http请求都会携带cookie，所以cookie只适合保存很小的数据，如会话标识。sessionStorage和localStorage 虽然也有存储大小的限制，但比cookie大得多，可以达到5M或更大。
    3. 数据有效期不同，sessionStorage：仅在当前浏览器窗口关闭前有效，自然也就不可能持久保持；localStorage：始终有效，窗口或浏览器关闭也一直保存，因此用作持久数据；cookie只在设置的cookie过期时间之前一直有效，即使窗口或浏览器关闭。
    4. 作用域不同，sessionStorage不在不同的浏览器窗口中共享，即使是同一个页面；localStorage 在所有同源窗口中都是共享的；cookie也是在所有同源窗口中都是共享的。
    5. Web Storage 支持事件通知机制，可以将数据更新的通知发送给监听者。
    6. Web Storage 的 api 接口使用更方便。

/**
 * 设置Cookie
 * key { String }：传入键
 * val { String }：传入值
 * day { Number }：有效期；单位天
*/
function setCookie (key, val, day) {
  const exDate = new Date()
  exDate.setDate(exDate.getDate() + day)
  document.cookie = `${key}=${escape(val) + (day == null ? '' : `;expires`${exDate.toGMTString()})}`
}
var token = 'asds4145451211122-41';
setCookie('token',token,1)

//获取cookie
function getCoookie (key) {
  let start = document.cookie.indexOf(key + '=')
  if (document.cookie < 0 || start !== -1) return '';
  start += ++key.length
  let end = document.cookie.indexOf(';', start)
  if (end == -1) end = document.cookie.length;
  return unescape(document.cookie.substring(start, end))
}
var token = getCookie('token');
alert(token)

function delCookie (key) {
  const exp = new Date();
  exp.setTime(exp.getTime() - 1);
  const cval = getCookie(key);
  if(cval != null) document.cookie = `${key}=${cval};expires=${exp.toGMTString()}`
}

call、apply的作用、区别

  • 作用：返回对应函数并立即调用
  • 相同点：
    • 修改函数上下文、函数都可以执行一次等
    • 都可以用于代替另一个对象调用一个方法，将一个函数的对象上下文从初始上下文改为 thisObject指定的新对象
  • 不同点：
    • 函数传参方式
    • call：
      • 直接的参数列表，主要用在js对象方法互相调用时，使当前this实例指向保持一致，或在特殊情况下需要改变this指向；若未提供thisObject参数，则全局对象被用作thisObject
    • apply：
      • 最多存在两个参数：新this对象、数组argArray
      • 若给该方法传递多个参数，即使只有一个参数也要写入数组内
      • 若argArray不是一个有效数组或不是内置对象将导致类型错误
      • 若未提供thisObject和数组的任一参数，将导致全局对象被用作thisObject且无法传递任何参数
    • demo
      function fun (a, b) { console.log(this, a, b) }
fun.apply({name: 'apply'}, [1,2,3,4,5,6,7])
/* {name: "apply"}, 1, 2 */
fun.call({name: 'call'}, 'hello', 'world')
/* {name: "call"}, hello, world */

一个页面从输入URL到页面加载显示完成，这个过程中都发生了什么？描述的越详细越好

  1. 浏览器地址栏输入url
  2. 浏览器依次查看浏览器缓存、系统缓存、路由缓存；如果存在则直接显示，否则下一步
  3. 域名解析（DNS）获取相应IP
  4. 浏览器向服务器发起TCP链接，与浏览器建立TCP三次握手
  5. 握手成功，浏览器向服务器发起Http请求，请求数据包
  6. 服务器请求数据，将数据返回浏览器
  7. 浏览器接受响应，读取页面内容，解析html源码，生成DOM树
  8. 解析css样式，浏览器渲染，js交互绑定多个域名，不限数量
  • 三次握手
    • 知道了服务器的 IP 地址，下面便开始与服务器建立连接了。
    • 通俗地讲，通信连接的建立需要经历以下三个过程：
      1. 主机向服务器发送一个建立连接的请求（您好，我想认识您）；
      2. 服务器接到请求后发送同意连接的信号（好的，很高兴认识您）；
      3. 主机接到同意连接的信号后，再次向服务器发送了确认信号（我也很高兴认识您），自此，主机与服务器两者建立了连接。
    • *补充说明
      • TCP 协议：
        • 三次握手的过程采用 TCP 协议，其可以保证信息传输的可靠性，三次握手过程中，若一方收不到确认信号，协议会要求重新发送信号。
      • 网页请求与显示
        • 当服务器与主机建立了连接之后，下面主机便与服务器进行通信。网页请求是一个单向请求的过程，即是一个主机向服务器请求数据，服务器返回相应的数据的过程。
        • 浏览器根据 URL 内容生成 HTTP 请求，请求中包含请求文件的位置、请求文件的方式等等；
        • 服务器接到请求后，会根据 HTTP 请求中的内容来决定如何获取相应的 HTML 文件；
        • 服务器将得到的 HTML 文件发送给浏览器；
        • 在浏览器还没有完全接收 HTML 文件时便开始渲染、显示网页；
        • 在执行 HTML 中代码时，根据需要，浏览器会继续请求图片、CSS、JavsScript等文件，过程同请求 HTML ；
  • 四次挥手
    • 主机向服务器发送一个断开连接的请求（不早了，我该走了）；
    • 服务器接到请求后发送确认收到请求的信号（知道了）；
    • 服务器向主机发送断开通知（我也该走了）；
    • 主机接到断开通知后断开连接并反馈一个确认信号（嗯，好的），服务器收到确认信号后断开连接；
    • *补充说明
      • 为什么服务器在接到断开请求时不立即同意断开：
        • 当服务器收到断开连接的请求时，可能仍然有数据未发送完毕，所以服务器先发送确认信号，等所有数据发送完毕后再同意断开。
      • 第四次握手后，主机发送确认信号后并没有立即断开连接，而是等待了 2 个报文传送周期，原因是：
        • 如果第四次握手的确认信息丢失，服务器将会重新发送第三次握手的断开连接的信号，而服务器发觉丢包与重新发送的断开连接到达主机的时间正好为 2 个报文传输周期。
